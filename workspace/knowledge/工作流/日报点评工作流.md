# 日报点评工作流标准

> 生成日期：2026-03-18
> 岗位：总经理助理

---

## 一、概述

对员工每日提交的日报进行整理、分析并点评，反馈给员工。

### 执行时间
| 项目 | 说明 |
|------|------|
| 执行时间 | 每天上午 10:00（北京时间） |
| 生效日期 | 仅限中国法定工作日（节假日不执行） |
| 触发方式 | 自动触发 |

### 数据范围
| 数据类型 | 说明 | 数据来源 |
|----------|------|----------|
| 昨日日报原文 | 必须引用 | 飞书多维表格（日报汇总表） |
| 近10天历史日报 | 作为对比参考 | 飞书多维表格（日报汇总表） |
| 昨日销售晨会内容 | 仅销售岗位（非周一） | "销售顾问沟通群"群聊记录 |

### 特殊说明（双休）
| 情况 | 处理方式 |
|------|----------|
| 周一 | 获取周五日报（周例会无晨会） |
| 周二 | 不获取周一晨会（周一无晨会） |
| 其他工作日 | 正常获取昨日日报+晨会 |

### 核心原则
| 原则 | 说明 |
|------|------|
| 鼓励与鞭策各占一半 | 点评中肯定与批评内容各占50% |
| 实事求是 | 以日报原文为依据 |
| 关注计划完成度 | 对比前天计划 vs 昨日完成 |
| 肯定日报颗粒度 | 从日报细致程度肯定员工 |
| 晨会与日报一致性 | 销售人员要对比晨会内容与日报内容 |

## 二、工作流程

| 步骤 | 步骤名称 | 操作内容 |
|------|----------|----------|
| 第1步 | 获取日报数据 | 登录飞书多维表格（日报汇总表），获取"汇报对象"包含姚霖的昨日记录 |
| 第2步 | 获取历史数据 | 获取前天写的明日计划、昨日写的今日总结、近10天历史日报 |
| 第2.1步 | 销售晨会对比 | 获取"销售顾问沟通群"晨会内容，对比承诺与完成情况（仅销售岗位） |
| 第3步 | 整理日报内容 | 直接引用原文，包含：今日总结、明日计划、经验沉淀、需要帮助 |
| 第4步 | 分析要点 | 按4.1-4.6进行分析 |
| 第5步 | 撰写点评 | 按5.1-5.4结构撰写 |
| 第6步 | 发送点评 | 按发送流程执行，先确认后发送 |

### 第1步：获取日报数据
- 登录飞书多维表格（日报汇总表）
- 获取"汇报对象"包含姚霖的记录
- 筛选昨日提交的日报

### 第2步：获取历史数据
- 获取员工"前天写的明日计划"
- 获取员工"昨天写的今日总结"
- 获取近10天历史日报，对比趋势
- 销售人员额外获取：昨日销售晨会内容、今日销售晨会内容
- 对比计划与实际的完成情况

### 第2.1步：销售晨会内容对比（仅销售岗位）
- 提取昨日晨会中销售人员承诺当日要达成的结果
- 提取今日晨会中销售人员承诺当日要达成的结果
- 对比晨会承诺与日报实际完成情况
- 找出"晨会说要做到但没做到"的事项

### 第3步：整理日报内容
- **直接引用**员工原文，不删减
- 必须包含四个部分：
  1. 今日总结
  2. 明日计划
  3. 经验沉淀（如有）
  4. 需要帮助（如有）

### 第4步：分析要点

#### 4.1 计划完成度分析
| 分析项 | 说明 |
|--------|------|
| 对比内容 | "前天写的明日计划" vs "昨天写的今日总结" |
| 分析要点 | 找出哪些计划完成了、哪些没完成 |
| 未完成处理 | 要在点评中询问原因 |

#### 4.2 销售晨会对比分析（仅销售岗位）
| 分析项 | 说明 |
|--------|------|
| 数据来源 | "销售顾问沟通群"群聊记录 |
| 对比内容 | 晨会承诺 vs 日报实际完成情况 |
| 分析要点 | 找出"晨会说要做到但没做到"的事项 |

#### 4.3 工作量分析
| 岗位类型 | 关注重点 |
|----------|----------|
| 销售人员 | 拜访量是否≥3次 |
| 非销售人员 | 工作是否饱和 |

#### 4.4 明日计划合理性分析
| 分析项 | 说明 |
|--------|------|
| 计划质量 | 是否具体、可执行 |
| 任务量 | 是否合理 |
| 时间节点 | 是否有明确的截止时间 |

#### 4.5 经验沉淀分析
| 情况 | 处理方式 |
|------|----------|
| 写了 | 要肯定（复盘意识好） |
| 未写 | **要指出**，提醒每天要复盘 |

#### 4.6 近10天趋势分析
| 分析项 | 说明 |
|--------|------|
| 计划完成率 | 是否有改善 |
| 工作量趋势 | 是否稳定 |
| 持续问题 | 是否有连续未完成的计划 |

---

### 第5步：撰写点评

#### 5.1 点评结构

**给用户确认时（包含原文引用）**：
```
【日报点评-确认】
━━━━━━━━━━━━━━━━━━━━━━━━━━
员工：XXX | 部门：XX部 | 职务：XXX

【原文引用】
昨日计划：xxx
昨日完成：xxx
经验沉淀：xxx（如有）
需要帮助：xxx（如有）

【对比分析】
前天计划：x项
昨日完成：x项
计划完成度：x%（x/x）
新增成果：（如有）
**销售晨会对比**（仅销售岗位）：
  - 昨日晨会承诺：xxx
  - 实际完成：xxx
  - 差距：xxx

💡 【做得好的地方】
- [具体肯定事项]

⭐ 【需要改进的地方】
- [具体问题]

📌 【建议】
- [改进方向]
━━━━━━━━━━━━━━━━━━━━━━━━━━
请确认是否发送
```

**发送给员工时（无需引用原文）**：
```
【日报点评】
━━━━━━━━━━━━━━━━━━━━━━━━━━
XXX，昨天的日报已查阅。

💡 【做得好的地方】
- [内容与确认版完全一致]

⭐ 【需要改进的地方】
- [内容与确认版完全一致]

📌 【建议】
- [内容与确认版完全一致]
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 5.2 鞭策要点（符合任一条件即需鞭策）
| 序号 | 鞭策条件 | 数据来源 |
|------|----------|----------|
| 1 | 日报颗粒度不够细致 | 当日日报 |
| 2 | 计划内容未完成 | 前天计划 vs 昨日完成 |
| 3 | **销售晨会承诺未达成** | 销售顾问沟通群 |
| 4 | 工作量不饱和 | 完成事项统计 |
| 5 | 销售人员拜访量＜3次 | 当日日报 |
| 6 | 明日计划不够具体明确 | 当日日报 |
| 7 | **未写经验沉淀** | 当日日报 |
| 8 | 进行与本职工作无关的事情或低价值工作 | 当日日报 |

#### 5.3 表扬要点（符合任一条件即需表扬）
| 序号 | 表扬条件 | 数据来源 |
|------|----------|----------|
| 1 | 成功达成计划目标 | 前天计划 vs 昨日完成 |
| 2 | 日报颗粒度细致、详实 | 当日日报 |
| 3 | **工作饱和度有提升空间** | 完成事项统计（不要说饱和，要说还有提升空间） |
| 4 | 销售人员拜访量≥3次 | 当日日报 |
| 5 | 有经验沉淀（复盘意识好） | 当日日报 |

#### 5.4 撰写原则
| 原则 | 说明 |
|------|------|
| **鼓励鞭策各占一半** | 肯定与批评内容各占50% |
| **点评要翔实** | 认真思考，每项分析都要有具体展开，有依据让人信服 |
| 语气温和 | 鞭策时对事不对人 |
| 事实为依据 | 不说不被信服的话，引用原文 |
| 引用原文 | 以日报原文为依据，不删减 |
| 问题具体化 | 每个问题都要有具体事实支撑 |
| **不说工作量饱和** | 要说"还有提升空间"，引导员工更努力 |

---

### 第6步：发送点评

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 给用户确认 | 展示包含原文引用的完整点评 |
| 2 | 用户确认 | 用户说"发送"后执行 |
| 3 | 发送给员工 | **直接使用确认版内容，仅删除"原文引用"和"对比分析"区块，其他内容完全一致** |
| 4 | 检查结果 | 发送完成后检查每条结果 |
| 5 | 补发 | 失败的立即补发 |
| 6 | 汇报 | 全部完成后再汇报 |

## 五、常见问题处理

| 情况 | 处理方式 |
|------|----------|
| 未写经验沉淀 | **要指出**，提醒每天要复盘 |
| 未写需要帮助 | 不强制要求 |
| 计划未完成 | 询问原因，分析近10天趋势 |
| 工作量不饱和 | 指出并提醒 |
| 销售拜访<3次 | 重点提醒 |
| 明日计划不具体 | 建议明确具体目标 |
| 晨会承诺未达成（销售） | 重点鞭策，对比晨会内容与日报 |

## 六、注意事项

| 序号 | 注意事项 |
|------|----------|
| 1 | 用用户身份发消息前必须先确认 |
| 2 | 发消息格式：如下为道简网络AI助理进行的日报点评... |
| 3 | 发送后必须检查结果，失败立即重发 |
| 4 | 全部发送完成后再汇报 |
| 5 | 点评内容要翔实，不能过少 |
| 6 | 以日报原文为依据，不删减 |
| 7 | 鼓励鞭策各占一半 |

## 七、后续优化方向

| 序号 | 优化方向 | 说明 |

| 1 | 建立员工历史日报数据库 | 持续跟踪改进情况 |
