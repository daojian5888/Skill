## 前言

### 1. 文档目的

本手册是道简网络应用系统（DJAS）行政管理子系统（DJAS-XZ）的标准操作规范，是DJOS-XZ（行政管理操作系统，关注"如何做"）的配套应用层文档，专注于"用什么工具和系统执行行政管理"。本子系统覆盖办公系统管理、行政数据管理、行政流程自动化、行政报表系统、行政移动办公、行政系统运营六大核心模块，确保公司行政管理工作的工具化、系统化、智能化。

道简网络主营企业数字化解决方案、智能安防与弱电工程、系统集成、IT全周期运维服务，行政管理子系统旨在将传统行政管理流程数字化、自动化，提升管理效率，降低管理成本。

### 2. 适用范围

本手册适用于道简网络全体涉及行政管理的员工，覆盖以下管理场景：
- 办公OA系统、文档管理系统、协作工具的配置与管理
- 行政资产数据、供应商数据、费用数据的系统化管理
- 行政流程的自动化配置与执行
- 行政KPI看板、成本报表、资产报表的输出
- 移动审批、移动报表等移动办公场景
- 行政系统的日常运维和持续优化

### 3. 核心原则

| 序号 | 原则 | 说明 |
|------|------|------|
| 1 | **系统化思维** | 行政管理事务应尽可能纳入系统管理，减少线下操作 |
| 2 | **自动化优先** | 能自动化的流程不手动处理，提升效率 |
| 3 | **数据驱动** | 基于系统数据进行决策，而非经验判断 |
| 4 | **用户体验** | 系统设计须考虑用户体验，操作简便易用 |
| 5 | **持续迭代** | 行政系统须持续优化迭代，适应业务变化 |

## 第1章 概述

### 1.1 定位与职责

#### 1.1.1 行政管理子系统定位

行政管理子系统（DJAS-XZ）是道简网络运行系统（DJRS）应用层（DJAS）的支撑性子系统，专注于行政管理的工具化和系统化，关注"用什么系统和工具执行行政管理"。本子系统与DJOS-XZ（流程标准）配套，DJOS-XZ定义"如何做"，DJAS-XZ定义"用什么工具做"，共同构成完整的行政管理解决方案。

#### 1.1.2 行政管理职责矩阵

| 角色 | 主要职责 | 关键系统 |
|----------|----------|----------|
| **系统管理员** | OA系统配置、权限管理、数据维护 | OA系统后台 |
| **行政专员** | 日常行政事务处理、系统操作 | 资产系统、费控系统 |
| **行政负责人** | 行政决策、系统优化需求提出 | 行政报表系统 |
| **全体员工** | 使用行政系统完成日常事务 | OA、考勤、资产等 |

### 1.2 模块架构

#### 1.2.1 模块总览

| 模块编号 | 模块名称 | 核心职能 |
|----------|----------|----------|
| 模块一 | 办公系统管理 | OA系统、文档系统、协作工具的配置与运维 |
| 模块二 | 行政数据管理 | 资产数据库、供应商数据库、费用数据库 |
| 模块三 | 行政流程自动化 | 审批流配置、表单设计、自动化规则 |
| 模块四 | 行政报表系统 | KPI看板、成本报告、资产报告 |
| 模块五 | 行政移动办公 | 移动审批、移动报表、位置服务 |
| 模块六 | 行政系统运营 | 系统配置、用户管理、数据备份、持续改进 |

#### 3.3.3.2 审批流程配置规范

| 配置项 | 配置要求 | 配置标准 |
|--------|---------|---------|
| **流程名称** | 简洁明确 | "XX申请流程" |
| **审批节点** | 节点数量合理 | 不超过5个节点 |
| **审批人** | 明确到岗位 | 不写具体人名，写岗位 |
| **审批规则** | 清晰可执行 | 写明什么条件通过/拒绝 |
| **字段必填** | 合理设置 | 仅必要的字段设为必填 |
| **字段权限** | 按需设置 | 审批人可查看必要字段 |
| **超时提醒** | 设置提醒 | 审批超过24小时自动提醒 |
| **异常处理** | 设置代理 | 审批人不在时设置代理审批 |
