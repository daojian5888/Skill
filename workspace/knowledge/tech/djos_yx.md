# 3.3-DJOS-YX 营销业务子系统标准操作流程

> **文档版本**：V1.0  
> **生效日期**：2026年3月  
> **编制部门**：道简网络销售部  
> **适用范围**：道简网络全体营销人员（销售部、商务部门）及管理人员

---

## 前言

### 1. 文档目的

本手册是道简网络操作系统（DJOS）营销业务子系统的标准操作规范，旨在为营销团队提供统一、可执行、可量化的作业标准。通过规范化日常行为管理、线索管理、客户管理、项目管理、报价管理、合同管理、订单与回款管理七大核心模块，确保营销业务规范化运作，支撑道简网络销售目标的持续达成。

### 2. 适用范围

本手册适用于道简网络全体营销人员（含销售顾问、销售经理、商务经理、产品经理），覆盖以下业务场景：

- 日常考勤、外勤管理
- 线索获取、分配、跟进与转化
- 客户开发、分类与日常维护
- 项目信息录入、变更与报备
- 报价单制作、审核与发出
- 合同起草、审批、签署与归档
- 订单处理、发货跟踪与回款管理

### 3. 核心原则

| 序号 | 原则 | 说明 |
|------|------|------|
| 1 | **客户为中心** | 所有营销行为以客户需求为导向，以成交为目标 |
| 2 | **数据驱动** | 所有业务动作须在系统中留痕，用数据指导决策 |
| 3 | **合规经营** | 所有营销行为须符合公司制度及行业法规 |
| 4 | **高价值优先** | 资源投入优先保障高价值客户和高概率项目 |
| 5 | **廉洁自律** | 严禁任何形式的商业贿赂和利益输送 |

## 模块一：日常行为管理

### 1.1 目的和适用范围

**目的**：建立规范的日常行为管理体系，确保营销团队高效工作、合规展业，为客户提供专业的服务形象，同时为公司经营分析提供真实的数据支撑。

**适用范围**：全体营销人员的日常签到、日报填写、夕会参与、周例会参与及外勤管理等场景。

### 1.2 日常考勤管理

#### 1.2.1 考勤方式与规则

| 考勤方式 | 适用范围 | 操作规范 |
|----------|----------|----------|
| **飞书打卡** | 坐班日（开会、培训） | 上班/下班各打卡1次 |
| **外勤定位打卡** | 外出拜访客户 | 定位+拍照上传（拜访现场） |
| **补卡申请** | 因公无法打卡 | 2日内提交+直属上级证明 |

#### 1.2.2 考勤违规处理

| 违规类型 | 处理规则 | 月额度 |
|----------|----------|--------|
| 忘打卡 | 补卡申请+直属上级证明 | 3次/月 |
| 迟到/早退（30分钟内） | 20元/次 | 无上限 |
| 旷工 | 无薪+绩效扣减 | 0容忍 |

### 1.3 日报管理

#### 1.3.1 日报填写规范

| 项目 | 要求 | 截止时间 |
|------|------|----------|
| **今日总结** | 填写当日主要工作及成果，至少3项 | 当日24:00前 |
| **明日计划** | 填写次日工作计划，至少3项 | 当日24:00前 |
| **经验沉淀** | 分享当日工作心得、问题或方法论 | 当日24:00前 |
| **需要帮助** | 如有需协助事项填写 | 当日24:00前 |

#### 1.3.2 日报考核规则

| 考核维度 | 考核标准 |
|----------|----------|
| 按时提交率 | 100%（迟交视为未提交） |
| 内容完整性 | 四项内容全部填写，不完整扣分 |
| 内容质量 | 空洞笼统扣分，有具体数据和成果加分 |

### 1.4 夕会管理

#### 1.4.1 夕会时间与参与要求

```

### 1.6 外勤管理

#### 1.6.1 有效外勤认定标准

| 认定条件 | 说明 |
|----------|------|
| 精确地理位置 | 须在外勤打卡时定位到客户所在地 |
| 现场照片 | 须上传拜访现场照片（客户办公环境/会客室等） |
| 客户联系人 | 须有客户方人员参与沟通 |
| 详细沟通记录 | 须在系统中填写沟通内容摘要 |

#### 1.6.2 外勤考核规则

| 考核项目 | 目标 | 奖金 |
|----------|------|------|
| 月度满分外勤次数 | 50次/月 | 1000元 |
| 外勤合格率 | 100% | -- |

#### 1.6.3 外勤统计规则

| 有效外勤 | 无效外勤（不计入） |
|----------|-------------------|
| 客户现场拜访，有详细沟通记录 | 电话/视频/微信沟通 |
| 客户招标现场 | 在客户楼下/附近打卡 |
| 客户政府机关/项目现场 | 重复拜访同一客户（当日第二次起） |

### 1.7 执行标准与要求

1. **外勤真实性**：外勤打卡须为真实拜访现场，发现虚假外勤视为严重违规。
2. **日报及时性**：日报须在当日24:00前提交，迟到提交视为未提交。
3. **会议参与率**：夕会/周例会须全程参与，确有外出须提前请假并线上参加。
4. **数据完整性**：所有客户沟通须在系统中留有记录，口头沟通不算。

## 模块二：线索管理

### 2.1 目的和适用范围

**目的**：建立统一的线索全生命周期管理流程，确保线索资源得到高效转化，通过规范线索分配、跟进、转化与回收机制，最大化线索利用率，为销售业绩增长提供充足的弹药。

**适用范围**：所有来源的市场线索（公司资源、协会信息、线上数据、客户转介绍、厂商信息、市场活动、个人挖掘）的管理场景。

### 2.2 线索来源分类

| 来源类型 | 获取方式 | 线索特点 | 跟进优先级 |
|----------|----------|----------|------------|
| **公司资源** | 公司分配、市场活动收集 | 数量大，质量参差 | 中 |
| **协会信息** | 行业协会、商会活动 | B端为主，决策链长 | 高 |
| **线上数据** | B2B平台、官网留资 | 量大但响应率低 | 低 |
| **客户转介绍** | 已有客户推荐 | 信任度高，成单率高 | 最高 |
| **厂商信息** | 厂商渠道反馈 | 技术对接为主 | 中 |
| **市场活动** | 展会、沙龙、宣讲会 | 现场意向较明确 | 高 |
| **个人挖掘** | 销售自行开发 | 依赖个人能力 | 按人评估 |

### 2.3 线索分配规则

#### 2.3.1 分配原则

| 原则 | 说明 |
|------|------|
| **公平原则** | 线索按规则轮询分配，保证公平 |
| **能力匹配** | 根据线索类型匹配相应能力销售 |
| **区域就近** | 同城/就近优先分配 |
| **经验优先** | 高价值线索优先分配给资深销售 |

#### 2.3.2 分配规则

| 线索类型 | 分配规则 | 负责人 |
|----------|----------|--------|
| 公司统一线索 | 每周一由销售经理按规则统一分配 | 销售经理 |
| 客户转介绍 | 优先分配给转介绍来源人 | 销售经理 |
| 市场活动线索 | 按活动参与情况分配 | 销售经理 |
| 主动开发线索 | 销售自行管理，不统一分配 | 销售人员 |

#### 2.3.3 线索分配上限

| 销售人员级别 | 持有线索上限 |
|--------------|-------------|
| 初级销售顾问 | 30条 |
| 中级销售顾问 | 40条 |
| 高级销售顾问 | 50条 |

### 2.4 线索跟进流程

#### 2.4.1 线索跟进时效要求

| 跟进节点 | 时效要求 | 说明 |
|----------|----------|------|
| **首次联系** | 分配后72小时内 | 必须电话/微信联系一次 |
| **持续跟进** | 每周至少1次 | 有意向客户增加频次 |
| **无意向处理** | 3次无法联系 | 标注"待联系"，降级管理 |
| **长期无进展** | 15天未转化 | 退回线索池重新分配 |

#### 2.4.3 线索状态定义

| 状态 | 定义 | 系统标注 |
|------|------|----------|
| **新线索** |刚分配的线索 | 蓝色 |
| **跟进中** | 已首次联系，持续跟进 | 绿色 |
| **有意向** | 客户有明显采购意向 | 黄色 |
| **待定** | 需要持续观察，暂不投入 | 灰色 |
| **已转化** | 已转化为正式客户 | -- |
| **退回线索池** | 15天未转化退回 | -- |

### 2.5 线索转化与回收

#### 2.5.1 转化标准

| 转化条件 | 说明 |
|----------|------|
| 客户有明确需求 | 客户表达明确采购意向 |
| 客户有预算 | 客户有预算或愿意讨论预算 |
| 客户有时间节点 | 客户有计划实施时间 |
| 联系人有决策权 | 联系人是关键决策人 |

#### 2.5.2 回收机制

| 回收条件 | 回收规则 |
|----------|----------|
| 15天未转化 | 退回线索池，重新分配 |
| 30天未转化 | 永久退回线索池，领取人不得再次领取 |
| 3次无法联系 | 降级为"待联系"，减少跟进频次 |
| 连续2次回收 | 暂停该来源线索分配资格 |

### 2.6 执行标准与要求

1. **72小时必须触达**：分配线索后72小时内必须首次联系，否则收回重新分配。
2. **跟进记录完整**：每次跟进须在系统记录，空白跟进不计入有效跟进。
3. **转化才能领取新线索**：如持有线索未转化，暂不分配新线索（老客户除外）。
4. **禁止抢单**：线索分配后归领取人所有，其他人不许抢夺，违者重罚。
5. **禁止私藏线索**：所有线索须录入系统，私藏线索视为严重违规。

## 模块三：客户管理

### 3.1 目的和适用范围

**目的**：建立完善的客户分级与维护管理体系，通过差异化的客户服务策略，确保有限资源向高价值客户倾斜，同时维护好全量客户关系，支撑客户长期价值最大化。

**适用范围**：所有已转化客户的分级管理、客户信息维护、服务群管理、客户拜访管理等场景。

### 3.2 客户分级标准

#### 3.2.1 客户分级定义

| 级别 | 定义 | 上限数量 | 拜访频率 | 服务资源 |
|------|------|----------|----------|----------|
| **战略客户** | 年收入贡献≥50万，或战略意义重大 | 10家 | 每两周1次 | 最高优先级 |
| **重要客户** | 年收入贡献10-50万 | 20家 | 每月1次 | 高优先级 |
| **普通客户** | 年收入贡献1-10万 | 50家 | 每季度1次 | 中优先级 |
| **低价值客户** | 年收入贡献<1万 | 20家 | 谨慎投入 | 按需服务 |

#### 3.2.2 客户分级调整

| 调整类型 | 调整条件 | 调整审批 |
|----------|----------|----------|
| 升级 | 连续2年贡献增长≥30% | 销售经理+综合管理部 |
| 降级 | 连续1年无交易或贡献下降≥50% | 销售经理 |
| 新增 | 首次成交即按成交额定级 | 系统自动 |

### 3.4 公海客户管理

#### 3.4.1 公海客户定义

| 公海类型 | 定义 | 释放规则 |
|----------|------|----------|
| **无主公海** | 未分配给任何销售的客户 | 销售可自行领取 |
| **回收公海** | 180天未成交被系统收回的客户 | 按规则重新分配 |

#### 3.4.2 公海客户领取规则

| 条件 | 规则 |
|------|------|

### 3.5 客户服务群管理

#### 3.5.1 服务群建立要求

#### 3.5.2 服务群运营规范

| 规范项目 | 说明 |
|----------|------|
| 群名规范 | "XX公司-道简网络服务群" |
| 群公告 | 须有服务范围+紧急联系人 |
| 发言规范 | 工作时间4小时内必须回复 |
| 禁言规范 | 禁止发布与工作无关的内容 |
| 解散规范 | 客户明确结束合作后可解散 |

### 3.6 客户拜访管理

#### 3.6.1 拜访要求

| 客户级别 | 月度拜访要求 | 最低拜访次数 |
|----------|-------------|-------------|
| 战略客户 | 每两周1次 | 2次/月 |
| 重要客户 | 每月1次 | 1次/月 |
| 普通客户 | 每季度1次 | 1次/季度 |
| 低价值客户 | 按需拜访 | 按需 |

#### 3.6.2 拜访准备

| 准备项 | 内容 |
|--------|------|
| 客户背景 | 最新合作情况+近期动态 |
| 拜访目的 | 本次拜访要达成的目标 |

| 预约确认 | 提前1天与客户联系人确认 |

#### 3.6.3 拜访记录

| 必填字段 | 内容 |
|----------|------|
| 拜访时间 | 精确到分钟 |
| 拜访地点 | 客户现场地址 |
| 拜访对象 | 客户方参与人员姓名+职位 |
| 拜访内容 | 沟通的主要议题和内容摘要 |
| 拜访成果 | 达成的共识/下一步计划 |
| 客户需求变化 | 客户需求/预算/时间节点是否有变化 |

### 3.7 执行标准与要求

1. **客户100%建档**：所有成交客户须在系统中建立完整档案，档案不完整视为客户管理失职。
2. **服务群必须建**：战略客户和重要客户必须建立服务群，未建群视为客户管理不合格。
3. **拜访留痕**：每次拜访须在外勤系统中记录，不记录不算有效拜访。
4. **公海及时认领**：公海客户被领取后须在7天内首次联系，否则收回。
5. **客户信息更新**：客户信息变化须在3天内更新系统，保持信息准确性。

### 5.1 目的和适用范围

**适用范围**：所有销售项目中涉及的产品选型、报价单制作、报价审核、报价发出及报价管理场景。

### 5.4 报价毛利要求

#### 5.4.1 毛利分级标准

| 项目类型 | 最低毛利要求 | 说明 |
|----------|-------------|------|
| **纯货物** | ≥5% | 才计算提成 |
| **含施工** | ≥10% | 含人工/辅料 |
| **含客户费用** | ≥10% | 含客户方承担费用 |
| **纯服务** | ≥30% | 人工为主 |

#### 5.4.2 毛利审批权限

| 毛利范围 | 审批人 | 说明 |
|----------|--------|------|
| 达标毛利 | 商务经理 | 直接审核通过 |
| 低于达标5%以内 | 商务经理+销售经理 | 双重审批 |
| 低于达标5-10% | 商务经理+销售经理+总经理 | 总经理特批 |
| 低于达标10%以上 | 不予报价 | 必须调整方案 |

### 5.5 选型规范

| 规范项 | 要求 |
|--------|------|
| **选型数量** | 同一需求选型不超过3种 |
| **品牌要求** | 二线及以上正规品牌，禁止白牌/三无 |
| **成本核算** | 以实际采购成本为准 |
| **库存折算** | 库存产品按每月1%折算 |

### 5.6 执行标准与要求

1. **时效第一**：报价须在规定时限内完成，超时视为报价管理失职。
2. **毛利红线**：低于最低毛利要求的报价须走特批，不允许擅自低价成交。
3. **报价留痕**：所有报价须录入系统，口头报价不算。
4. **成本保密**：报价成本信息严禁外泄，违者重罚。
5. **选型合规**：选型须符合公司品牌要求，禁止推荐白牌/三无产品。

4. **履约跟踪**：合同签署后须跟踪履约情况，发现风险及时预警。
5. **档案完整**：合同签署后5个工作日内完成归档，缺失档案视为管理失职。
