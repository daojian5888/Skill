## 前言

### 1. 文档目的

本手册是道简网络操作系统（DJOS）行政子系统的标准操作规范，旨在为公司行政管理团队提供统一、可执行、可量化的作业标准。通过规范化组织架构与岗位管理、人事档案管理、考勤与休假管理、办公资产与设备管理、办公环境管理、后勤保障管理、行政审批流程及持续改进八大核心模块，确保公司行政运营高效有序，支撑道简网络整体经营目标的实现。

### 2. 适用范围

本手册适用于道简网络全体行政管理人员及各业务部门负责人，覆盖以下管理场景：

- 员工入转离全流程管理
- 考勤休假核算与薪酬对接
- 办公资产配置、保养与报废
- 办公环境安全与卫生管理
- 快递、印章、证照等后勤保障
- 行政类合同审批与费用管控
- 公司级行政活动组织

### 3. 核心原则

| 序号 | 原则 | 说明 |
|------|------|------|
| 1 | **制度先行** | 所有行政行为须有章可循，严禁随意突破制度 |
| 2 | **服务导向** | 行政工作的本质是为业务部门提供高效支持 |
| 3 | **成本意识** | 在保证服务品质的前提下，合理控制行政费用 |
| 4 | **合规安全** | 涉及印章、证照、合同等重要事项须严格合规 |
| 5 | **持续改进** | 通过数据分析与服务反馈，不断优化行政流程 |

### 1.5 执行标准与要求

1. **组织架构更新**：公司组织架构图如有调整，须在总经理审批通过后3个工作日内更新完毕。
2. **岗位说明书维护**：每个岗位须有对应的岗位说明书，内容包括：岗位职责、任职资格、考核指标。
3. **汇报关系明确**：每位员工须清楚自己的直接汇报对象，隔级汇报须知会直接上级。
4. **一岗多备**：关键岗位须有备份人员，确保人员离岗时业务不中断。

#### 2.3.2 试用期评估维度

| 评估维度 | 权重 | 评估内容 |
|----------|------|----------|
| **工作业绩** | 40% | 试用期工作目标完成情况 |
| **工作能力** | 30% | 专业技能、业务理解、学习能力 |
| **工作态度** | 20% | 责任心、主动性、团队协作 |
| **出勤纪律** | 10% | 试用期出勤情况 |

#### 2.3.3 转正结果分类

| 转正结果 | 条件 | 后续处理 |
|----------|------|----------|
| **按期转正** | 综合评分≥60分 | 正式转正，转正薪资生效 |

| **延长试用期** | 综合评分50-59分 | 延长试用期1个月，最多延长1次 |
| **不予转正** | 综合评分<50分 | 试用期内解除劳动合同 |

### 2.6 人事档案安全管理

| 安全等级 | 档案内容 | 访问权限 | 保管期限 |
|----------|----------|----------|----------|
| **高度机密** | 薪酬数据/身份证号/银行账户/病历 | 综合管理部经理+总经理 | 永久 |
| **机密** | 劳动合同/绩效考核/学历信息 | 综合管理部相关人员 | 离职后5年 |
| **内部** | 岗位职责/岗位调整记录/培训记录 | 部门负责人+综合管理部 | 任职期间+2年 |
| **公开** | 员工姓名/部门/岗位/工号 | 全员 | 任职期间 |

### 2.7 执行标准与要求

1. **档案完整性**：员工入职30天内须完成全部档案材料归档，不得缺失关键材料（身份证、学历证、劳动合同）。
2. **信息准确性**：员工个人信息变更须在5个工作日内提交书面申请，综合管理部审核后更新系统。
3. **档案保密性**：人事档案仅限综合管理部经授权人员访问，跨部门查阅须经综合管理部负责人批准。
4. **离职及时性**：员工离职后须在离职当日完成全部系统权限关闭，最迟不超过24小时。
5. **交接完整性**：未完成交接清单全部项目签字确认前，不得办理最后薪资结算。

## 模块三：考勤与休假管理

### 3.1 目的和适用范围

**目的**：建立公正、透明、符合法规的考勤休假管理体系，准确核算员工出勤与休假情况，为薪酬核算提供可靠数据依据，同时保障员工的法定休息权利。

**适用范围**：全体员工（含试用期员工，不含兼职/外聘特殊约定者）的日常考勤、请假审批、出差管理、加班管理等场景。

### 3.2 考勤管理制度

#### 3.2.1 工作时间规定

| 岗位类型 | 工作时间 | 午休时间 | 弹性规则 |
|----------|----------|----------|----------|
| **职能岗位** | 8:30-17:30 | 12:00-13:00 | 无 |
| **销售外勤岗位** | 8:30-18:00 | 12:00-13:00 | 以客户拜访为准 |
| **运维值班岗位** | 轮班制 | 轮班制 | 按排班表执行 |
| **特殊岗位** | 经审批的特殊工时制 | 按审批执行 | 按审批执行 |

#### 3.2.2 考勤方式

| 考勤方式 | 适用范围 | 操作规范 |
|----------|----------|----------|
| **飞书打卡** | 全体坐班员工 | 上班/下班各打卡1次，间隔不少于4小时 |
| **外勤定位打卡** | 销售/运维外出人员 | 拜访地点定位+拍照上传 |
| **补卡申请** | 全体员工 | 因公无法打卡须2日内提交补卡申请 |
| **免打卡人员** | 高管/外勤/特殊约定 | 经审批的免打卡人员名单 |

#### 3.2.3 考勤异常处理

| 异常类型 | 处理规则 | 每月额度 |
|----------|----------|----------|
| **忘打卡** | 提交补卡申请+直属上级证明 | 3次/月 |
| **迟到** | 30分钟内记迟到，超过30分钟记旷工 | 无额度 |
| **早退** | 30分钟内记早退，超过30分钟记旷工 | 无额度 |
| **旷工** | 无薪资发放，连续3天或累计5天启动纪律处分 | 0容忍 |
| **出差忘打卡** | 提交出差证明+补卡申请 | 不计入忘打卡额度 |

### 3.3 休假管理制度

#### 3.3.1 假期类型

| 假期类型 | 法定/福利 | 天数 | 说明 |
|----------|-----------|------|------|
| **法定节假日** | 法定 | 按国家规定 | 元旦/春节/清明/劳动节/端午/中秋/国庆 |
| **带薪年休假** | 法定 | 司龄1-10年：5天；10-20年：10天；20年+：15天 | 按入职周年计算 |
| **婚假** | 法定 | 10天（晚婚加5天，按法规） | 须提供结婚证 |
| **产假** | 法定 | 158天（难产+15天，多胞胎+15天） | 须提供生育证明 |
| **陪产假** | 法定 | 15天 | 须提供出生证明 |
| **流产假** | 法定 | 怀孕不满4个月：15天；满4个月：42天 | 须提供医疗证明 |
| **丧假** | 法定 | 父母/配偶/子女：3天；祖父母/外祖父母/岳父母：1天 | -- |
| **病假** | 法定 | 按医疗期规定执行 | 须提供医院证明 |
| **事假** | 福利 | 无上限 | 每月不超过3天，超出扣绩效 |
| **工伤假** | 法定 | 按工伤认定结果执行 | 须提供工伤认定材料 |

#### 3.3.2 年休假计算规则

```
年休假天数 = （当年度在本公司工作日历天数 / 365） × 应享年休假天数

不满1年折算：<12个月不享受年休假
入职不满周年：按转正后实际出勤月数折算
离职时清零：未休年假不折现（法定要求除外）
```

### 3.7 执行标准与要求

1. **打卡真实性**：考勤打卡须为本人实际操作，发现代打卡视为严重违纪。
2. **请假提前性**：请假须提前申请，突发情况须在当天9:30前通知直属上级，事后补办手续。
3. **审批时效性**：请假审批须在24小时内完成，审批人外出须指定代理人。
4. **销假及时性**：请假定<3天>如提前结束，须在返回当日办理销假手续。
5. **数据保密性**：考勤及薪资数据仅限综合管理部相关人员访问，其他人员查询须经本人同意。
6. **加班事前审批**：加班须提前申请并获得审批，未审批的加班不计入加班工时。

### 4.6 执行标准与要求

1. **一人一机**：每位员工配置一台办公电脑，特殊情况须经综合管理部审批。
2. **资产标签化**：所有资产须粘贴资产标签，标签内容包括：资产编码、使用人、购置日期。
3. **维修优先内部**：能内部维修的不送修，能送修的不换新，节约维修成本。
4. **报废先鉴定**：任何资产报废前须经综合管理部技术鉴定，禁止私自处置。
5. **年度盘点必须**：每年12月须完成全公司资产大盘点，盘点结果报总经理。
6. **离职资产回收**：员工离职须归还全部公司资产，未归还前不予办理最后薪资结算。

### 5.4 空间规划与6S管理

#### 5.4.1 6S管理标准

| 6S | 标准要求 | 执行检查 |
|----|----------|----------|
| **整理（Seiri）** | 区分必需品与非必需品，及时清理非必需品 | 每日工位检查 |
| **整顿（Seiton）** | 必需品定置管理，明确标识，随用随取 | 每周部门自查 |
| **清扫（Seiso）** | 保持工作区域干净整洁，无灰尘/无污渍 | 每日清洁 |
| **清洁（Seiketsu）** | 将前3S标准化、制度化，形成规范 | 综合管理部巡检 |
| **素养（Shitsuke）** | 养成良好习惯，遵守规章制度 | 月度综合检查 |
| **安全（Safety）** | 消除安全隐患，保障人身和财产安全 | 每日安全巡查 |

#### 5.4.2 空间使用规范

| 区域 | 使用规范 | 违规处理 |
|------|----------|----------|
| **工位区** | 禁止堆放私人物品/食品/饮料（饮用水除外） | 第一次警告，第二次扣绩效 |
| **会议室** | 提前预约/按时使用/使用后恢复原状 | 第一次警告，第二次通报 |
| **公共区域** | 禁止吸烟/禁止大声喧哗/通道保持畅通 | 按公司制度处理 |
| **储物间** | 物品定置管理/禁止存放易燃易爆物品 | 立即整改 |

### 5.5 执行标准与要求

1. **安全第一**：任何时候安全都是第一优先级，发现安全隐患须立即报告并处置。
2. **卫生轮值**：各部门负责本区域卫生，综合管理部负责公共区域及整体监督。
3. **6S日常化**：6S管理不是大扫除，而是每日保持的工作习惯。
4. **应急响应**：发生安全事件须在第一时间报告综合管理部，重大事件须同时报告总经理。
5. **访客陪同**：访客须由被访人全程陪同，不得让访客单独在办公区域活动。

### 7.4 费用报销流程

#### 7.4.1 报销标准

| 费用类型 | 报销标准 | 票据要求 |
|----------|----------|----------|
| **差旅费** | 按出差补贴标准执行 | 发票+出差报告 |
| **交通费** | 实报实销（公共交通为主） | 发票 |
| **通讯费** | 限额报销（高管200/月，员工100/月） | 发票 |
| **招待费** | 单次≤500元（事先审批） | 发票+招待清单 |
| **办公费** | 实报实销 | 发票 |
| **培训费** | 事前审批后报销 | 发票+培训通知 |

#### 7.4.2 报销流程

### 7.5 执行标准与要求

1. **事前审批**：涉及费用发生的事项须提前审批，事后报销须说明原因。
2. **票据合规**：所有报销须有正规发票，发票内容须与实际消费一致。
3. **时效要求**：费用发生后须在30天内完成报销，逾期不予报销（特殊除外）。
4. **虚假报销零容忍**：发现虚假报销须全额退还并按公司制度处理，情节严重予以开除。
5. **审批时效**：各审批节点须在24小时内完成审批，审批人外出须指定代理人。

## 模块八：持续改进

### 8.1 目的和适用范围

**目的**：建立行政服务的持续改进机制，通过数据分析、客户反馈、流程复盘等方式，不断提升行政服务质量与效率，降低行政运营成本，支撑公司整体经营效率的提升。

**适用范围**：行政服务质量的评估与改进、行政流程的优化、行政成本的管控等场景。

### 8.2 服务质量评估

#### 8.2.1 评估维度

| 评估维度 | 权重 | 评估内容 |
|----------|------|----------|
| **响应时效** | 30% | 后勤请求响应时间/审批时效 |
| **服务质量** | 30% | 服务态度/专业度/问题解决率 |
| **成本控制** | 20% | 预算执行率/单次服务成本 |
| **合规性** | 20% | 流程合规率/制度执行率 |

#### 8.2.2 评估方式

| 评估方式 | 频率 | 实施方 | 说明 |
|----------|------|--------|------|
| **满意度调查** | 每季度 | 综合管理部 | 向全体员工发放问卷 |
| **流程审计** | 每半年 | 综合管理部 | 抽查行政流程执行情况 |
| **成本分析** | 每月 | 财务部+综合管理部 | 分析各项费用变动原因 |
| **服务复盘** | 每季度 | 综合管理部 | 复盘重大后勤事件 |

### 8.3 改进机制

#### 8.3.1 改进来源

| 改进来源 | 发现渠道 | 处理优先级 |
|----------|----------|------------|
| 员工反馈 | 满意度调查/日常反馈 | 高 |
| 审批超时 | 流程时效数据 | 高 |
| 成本超支 | 费用分析报告 | 高 |
| 合规问题 | 流程审计发现 | 高 |
| 行业对标 | 学习研究 | 中 |
| 最佳实践 | 外部学习交流 | 中 |

#### 8.3.2 改进分类与处理流程

| 改进类型 | 定义 | 处理流程 |
|----------|------|----------|
| **流程优化** | 简化审批环节/提升效率 | 评估→试点→全面推广 |
| **成本优化** | 降低行政费用/提升性价比 | 方案评估→供应商谈判→实施 |
| **服务创新** | 引入新服务/提升体验 | 需求分析→方案设计→试点→推广 |
| **制度建设** | 完善制度/堵塞漏洞 | 制度起草→会签→发布 |

### 8.4 执行标准与要求

1. **改进闭环**：所有改进事项须录入改进跟踪表，设置责任人、完成时限、验证方式。
2. **数据驱动**：行政决策须有数据支撑，禁止仅凭主观判断进行改进。
3. **成本效益平衡**：改进方案须进行成本效益分析，避免过度投入。
4. **员工参与**：鼓励员工提出改进建议，采纳的建议给予适当奖励。
5. **季度汇报**：综合管理部每季度向管理层汇报行政运营情况及改进进展。
