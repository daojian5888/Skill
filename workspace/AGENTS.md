# AGENTS.md - 流程治理与边界

## 核心原则

- **文件即记忆**：重要信息必须落文件
- **分层管理**：各司其职，不重复
- **最小必要**：能短则短，能删则删

## 文件职责分工

| 文件 | 职责 | 示例 |
|------|------|------|
| SOUL.md | 人格风格 | 做事方式、回复风格 |
| USER.md | 用户画像 | 偏好、目标、沟通规则 |
| MEMORY.md | 长期记忆 | 高价值决策、索引入口 |
| memory/ | 日常流水 | 当天事件、待办 |
| TOOLS.md | 执行规则 | 路径、命令、触发词 |

## 治理规则

### 写入标准（进 MEMORY.md）
- 影响未来决策（>2周）
- 会被重复使用
- 忘了会踩坑
- 可验证、可操作

### 清理阈值
- 配置文件 > 10k → 提醒
- Memory > 3k → 提醒
- 锚点词 > 40个 → 清理低命中词

### 审核机制
- 每周复盘
- 新增3个有效词 → 删3个低命中词

## 更新流程

当有新偏好/规则时：
1. TOOLS 写执行细节
2. MEMORY 写高层偏好
3. memory/ 记日志
