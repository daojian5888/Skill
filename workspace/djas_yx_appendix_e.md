## E.0 附录概述

### E.0.1 附录目的

本附录为DJAS-YX营销应用子系统的深度操作规程补充，旨在规范营销人员在日常业务开展过程中对各类营销工具的标准操作流程、注意事项及合规要求，确保营销业务规范化、标准化执行，防范业务风险，提升整体营销效能。

### E.0.2 适用范围

本规程适用于道简网络科技有限公司全体营销人员（销售部、商务部及其他涉及营销业务的部门人员），涵盖客户管理、商机跟进、数据分析、营销自动化、物料管理及系统安全等全链条营销工具操作场景。

### E.0.3 核心原则

| 原则 | 说明 |
|------|------|
| **数据真实性** | 所有营销数据必须真实记录、如实填报，严禁虚构、篡改业务数据 |
| **客户信息安全** | 严格保护客户商业信息，不得泄露、传播非公开客户数据 |
| **流程规范性** | 按本规程规定的操作步骤执行，不得随意简化或跳过必要流程 |
| **合规优先** | 在开展任何营销活动前，必须确认符合公司合规要求及法律法规 |

### E.0.4 工具分类总览

| 工具类别 | 工具名称 | 适用场景 | 使用频次 |
|----------|----------|----------|----------|
| 客户管理 | 飞书CRM系统 | 客户信息管理、商机跟踪、销售跟进 | 每日 |
| 数据分析 | 营销数据看板/BI工具 | 业绩统计、报表生成、数据提取 | 每周/每月 |
| 营销自动化 | 线索分配系统/跟进提醒工具 | 线索分配、任务提醒、自动化流程 | 每日 |
| 物料管理 | 营销物料管理系统 | 宣传物料申请、样品管理、库存盘点 | 按需 |
| 系统安全 | 权限管理与安全规范 | 账号管理、权限配置、安全审计 | 每月 |

## E.1 飞书CRM系统深度操作规程

### E.1.1 系统概述与登录规范

飞书CRM系统是道简网络营销业务的核心管理平台，用于集中管理客户信息、商机项目及销售跟进记录。所有营销人员必须通过企业飞书账号统一登录，严禁使用个人账号或他人账号操作CRM系统。

**登录规范要求**：

| 序号 | 规范要求 | 违规处理 |
|------|----------|----------|
| 1 | 仅使用企业配发的飞书账号登录CRM系统 | 违规使用他人账号立即冻结权限并通报 |
| 2 | 密码设置符合公司复杂度要求（8位以上，含大小写字母及数字） | 密码不合规系统拒绝注册并提示修改 |
| 3 | 离开工位时必须锁定CRM系统界面（Ctrl+Win+L快捷键） | 未经锁定离开被他人操作，责任人承担后果 |
| 4 | 定期更换登录密码（建议90天内更换一次） | 超期未更换被安全审计发现纳入考核 |
| 5 | 发现账号异常登录立即报告技术部门并更改密码 | 知情不报导致数据泄露从重处理 |

**系统登录操作步骤**：

**步骤1：打开飞书客户端**
在电脑端打开飞书应用程序，确保已更新至最新版本。点击左侧导航栏中的"工作台"入口。

**步骤2：进入CRM应用**
在工作台中找到"CRM"应用图标，点击进入。如工作台中未显示CRM应用，需联系系统管理员开通访问权限。

**步骤3：完成身份验证**
系统将自动识别当前登录的飞书账号。如需二次验证，按系统提示完成短信验证码或企业邮箱验证。

**步骤4：进入主工作台**
验证通过后进入CRM主界面，默认显示"我的客户"列表页面。核对右上角显示的用户信息与本人一致。

**注意事项**：
- 登录前确认电脑已安装公司认可的杀毒软件，禁止在公共电脑上登录CRM系统
- 如遇登录失败，先检查网络连接，再确认账号状态，必要时联系IT支持
- 移动端（手机）可通过飞书APP内嵌CRM轻应用进行日常查询和跟进记录，但敏感操作（删除、转移客户等）须在PC端完成

### E.1.2 客户管理深度操作规程

#### E.1.2.1 新建客户档案

所有潜在客户和成交客户均须在CRM系统中建立正式档案，不得在系统外进行客户信息记录。

**新建客户档案操作步骤**：

**步骤1：进入客户列表页面**
登录CRM系统后，点击左侧菜单"客户管理"，进入"全部客户"列表页。

**步骤2：点击新建客户**
在客户列表页右上角找到"+新建客户"按钮，点击打开新建客户表单。

**步骤3：填写客户基本信息**

| 字段名称 | 填写要求 | 示例 |
|----------|----------|------|
| 客户名称 | 全称（与营业执照一致），不得使用简称或代号 | 青岛某某科技有限公司 |
| 客户类型 | 单选：潜在客户/意向客户/成交客户/沉默客户 | 潜在客户 |
| 所属行业 | 从下拉列表选择，不在列表内选"其他"并备注 | 制造业 |
| 所在地区 | 精确到市辖区，如"山东省青岛市市北区" | 山东省青岛市市北区 |
| 企业规模 | 单选：微型企业/小微企业/中型企业/大型企业 | 小微企业 |
| 客户来源 | 单选：主动获客/电话营销/展会获客/老客推荐/平台引流/其他 | 主动获客 |
| 统一社会信用代码 | 18位代码，成交客户必填 | 91370200XXXXXXXXXX |

**步骤4：填写联系人信息**

| 字段名称 | 填写要求 |
|----------|----------|
| 联系人姓名 | 真实姓名，不得使用昵称 |
| 职务 | 如实填写，无职务则填"负责人" |
| 手机号 | 11位手机号，验证格式有效性 |
| 邮箱 | 验证格式有效性，优先填写工作邮箱 |
| 微信 | 如实填写，用于后续客情维护 |
| 是否为决策人 | 单选：是/否/不确定 |

**步骤5：填写商机信息（如适用）**

| 字段名称 | 填写要求 |
|----------|----------|
| 商机名称 | 格式：客户名称+项目类型，如"某某科技网络系统集成项目" |
| 预计签单金额 | 填写预估合同金额（万元），精确到小数点后两位 |
| 预计签单日期 | 填写预估合同签订日期，需在当前日期之后 |
| 商机阶段 | 单选：初步接触/需求确认/方案报价/商务谈判/合同签订 |
| 我方对接人 | 自动填充当前登录用户 |

**步骤6：保存并提交**
检查所有填写内容准确无误后，点击"保存"按钮。系统自动创建客户编号，完成新建流程。

**注意事项**：
- 客户名称、手机号、邮箱为必填字段，不完整无法保存
- 同一客户不得重复创建档案，如发现重复需合并或报告主管处理
- 商机阶段需根据实际业务进展及时更新，不得长期停留在初始阶段
- 新建客户后24小时内须完成首次跟进记录，否则系统自动预警

#### E.1.2.2 客户信息完善与维护

已建档客户的信息须保持最新状态，营销人员须在每次客户接触后及时更新相关信息。

**客户信息维护操作步骤**：

**步骤1：查询目标客户**
在客户列表页通过客户名称、手机号、联系人等关键词进行搜索，找到目标客户记录。

**步骤2：进入客户详情页**
点击客户名称或右侧"查看"按钮，进入客户详情页面。

**步骤3：更新联系信息**
检查并更新联系人的手机号、邮箱、职务等变动信息，注明"信息更新时间"。

**步骤4：记录跟进情况**
在"跟进记录"模块中点击"+新增跟进"，填写本次跟进详情：

| 跟进字段 | 填写说明 |
|----------|----------|
| 跟进方式 | 单选：电话/邮件/上门拜访/线上会议/微信/其他 |
| 跟进时间 | 系统自动记录，也可手动调整 |
| 跟进内容 | 详细记录本次跟进的核心内容和结果（不少于50字） |
| 下次跟进计划 | 如有，填写下次跟进时间和主要事项 |
| 商机金额变动 | 如签单金额有调整，在此更新并注明调整原因 |

**步骤5：更新客户标签**
根据最新了解到的客户需求特征，更新客户标签（如"重点客户""需尽快跟进""已报价待回复"等）。

**步骤6：保存更新**
确认所有信息填写完整后保存。

**注意事项**：
- 跟进记录是客户管理质量的核心考核依据，每次客户接触必须记录
- 跟进内容须客观真实反映沟通情况，不得编造或敷衍填写
- 客户手机号、邮箱等核心联系方式变更须在24小时内更新
- 长期（超过30天）未跟进客户系统将自动移入"沉默客户"池

#### E.1.2.3 客户转移与分配

**客户转移操作步骤**：

**步骤1：进入客户详情页**
在CRM系统中找到需要转移的客户记录，进入其详情页面。

**步骤2：点击转移按钮**
在客户详情页右上角找到"转移客户"按钮，点击打开转移确认页面。

**步骤3：选择接收人**
在弹出的转移人员列表中选择接收该客户的营销人员。可通过姓名或部门进行搜索筛选。

**步骤4：填写转移说明**
在转移备注中填写本次转移的原因说明（如"人员岗位调整""客户区域划分变更"等）。

**步骤5：确认转移**
点击"确认转移"按钮。系统将向接收人发送转移通知，同时记录转移日志。

**合规要求**：
- 客户转移须经转出方和接收方双方主管确认
- 成交客户的转移须经销售总监审批
- 不得私自转移客户，转移记录永久留存
- 转移完成后原跟进记录完整保留，由接收人继续维护

### E.1.3 商机管理深度操作规程

#### E.1.3.1 商机创建与阶段推进

商机是销售过程管理的核心单元，每个有效商机须对应真实的销售项目机会。

**商机创建操作步骤**：

**步骤1：在客户详情页创建**
进入目标客户详情页，点击"商机"模块中的"+新建商机"。

**步骤2：填写商机基本信息**

| 字段名称 | 填写要求 | 示例 |
|----------|----------|------|
| 商机名称 | 必填，格式：客户简称+项目关键词 | 某某科技-网络系统集成 |
| 项目类型 | 单选：网络系统集成/智能安防/IT运维/软件服务/综合解决方案/其他 | 网络系统集成 |
| 预计签单金额 | 必填，单位万元，精确到小数点后两位 | 35.00 |
| 预计签单日期 | 必填，日期须在当前日期之后 | 2026-05-31 |
| 竞争对手 | 如有竞争，填写竞争对手名称 | 某某集成商 |
| 胜算评估 | 单选：很有把握/有一定把握/不确定/难度较大 | 有一定把握 |
| 商机来源 | 单选：老客户新需求/新客户开发/原有项目追加/招投标项目/其他 | 新客户开发 |

**步骤3：选择商机阶段**
根据项目实际进展选择对应阶段：

| 阶段 | 定义 | 进入条件 |
|------|------|----------|
| 初步接触 | 与客户建立联系，了解基本需求 | 完成首次有效沟通 |
| 需求确认 | 完成客户需求调研，确认项目范围 | 提交需求调研报告 |
| 方案报价 | 完成方案设计并向客户提交报价 | 报价文件已发送客户 |
| 商务谈判 | 与客户就合同条款进行谈判 | 进入合同条款协商 |
| 合同签订 | 完成合同签署 | 合同已签章 |

**步骤4：关联项目文档**
将需求方案、报价文件、合同草案等相关文档上传至商机附件区，便于团队协同和过程追溯。

**步骤5：设置跟进行动计划**
在"行动计划"模块中创建下次跟进任务，设定提醒时间和具体事项。

**商机阶段推进操作步骤**：

**步骤1：评估阶段条件**
在推进商机至下一阶段前，对照阶段定义确认满足进入条件。

**步骤2：更新跟进记录**
在跟进记录中详细说明推进至下一阶段的核心进展和依据。

**步骤3：执行阶段推进**
点击"推进阶段"按钮，选择目标阶段，填写阶段推进说明。

**步骤4：确认阶段变更**
系统记录阶段变更时间和操作人，商机状态更新为新阶段。

**注意事项**：
- 商机阶段须真实反映项目进展，严禁虚报阶段套取业绩
- 预计签单金额须与实际报价一致，虚报金额纳入诚信考核
- 预计签单日期到期未签单须及时调整或标注原因
- 连续2个阶段推进间隔超过45天系统自动预警

#### E.1.3.2 商机赢单与输单管理

**商机赢单（合同签订）操作步骤**：

**步骤1：确认合同已签订**
在确认与客户完成合同签署后，进入对应商机详情页。

**步骤2：点击赢单按钮**
点击商机状态区的"赢单"按钮。

**步骤3：填写赢单信息**

| 字段名称 | 填写要求 |
|----------|----------|
| 实际签单金额 | 填写合同实际金额 |
| 合同编号 | 填写正式合同编号 |
| 合同签订日期 | 填写实际签订日期 |
| 项目启动日期 | 预计项目开工日期 |
| 备注 | 填写其他需要说明的信息 |

**步骤4：关联CRM客户**
系统自动将客户类型从"意向客户"变更为"成交客户"，商机状态更新为"已赢单"。

**商机输单操作步骤**：

**步骤1：确认商机失败**
在明确项目无法达成后（如客户选择竞争对手、客户项目取消等），进入商机详情页。

**步骤2：点击输单按钮**
点击商机状态区的"输单"按钮。

**步骤3：选择输单原因**

| 输单原因分类 | 具体原因选项 |
|--------------|---------------|
| 竞争对手因素 | 价格劣势/技术方案差距/品牌认知度低/服务能力不足 |
| 客户因素 | 客户项目暂停/预算削减/客户选择其他供应商/客户内部变动 |
| 我方因素 | 响应速度慢/方案不匹配/资源不足/人员变动 |
| 其他因素 | 不可抗力/政策原因/其他 |

**步骤4：填写输单说明**
详细说明输单的核心原因和改进建议（不少于100字）。

**步骤5：完成输单登记**
确认信息无误后提交，商机状态更新为"已输单"。

**合规要求**：
- 赢单信息须与合同原件一致，财务部门交叉验证
- 输单记录永久留存，作为竞争分析和经验沉淀的依据
- 输单后如客户有新需求，可在原商机下追加跟进或新建商机
- 虚假赢单（未实际签订合同标记为赢单）视为严重违规

### E.1.4 销售跟进深度操作规程

#### E.1.4.1 日常跟进计划制定

**每日跟进计划制定操作步骤**：

**步骤1：打开工作台首页**
登录CRM系统后默认进入工作台首页，查看"今日待办"和"我的客户"模块。

**步骤2：查看系统推送**
系统自动推送今日待跟进客户列表（包括上次标记的跟进任务和即将到期的商机）。

**步骤3：制定个人跟进计划**
根据系统推送和自主安排，在"今日计划"模块中新增当日跟进任务：

| 任务字段 | 填写说明 |
|----------|----------|
| 客户名称 | 从已有客户列表选择或新建 |
| 跟进事项 | 简述本次跟进的核心目标 |
| 预计时长 | 预估本次跟进所需时间（分钟） |
| 跟进方式 | 选择：电话/上门/线上/邮件 |
| 优先级 | 单选：高/中/低 |

**步骤4：设置提醒**
为每项跟进任务设置系统提醒（建议提前15分钟）。

**步骤5：执行并记录**
完成跟进后在对应任务中填写跟进结果，更新客户信息。

**注意事项**：
- 每日须制定并执行至少3次有效客户跟进（以有记录为准）
- 上门拜访须在跟进记录中备注拜访地点和时间
- 跟进计划不得为空，连续3个工作日无跟进计划纳入绩效考核

#### E.1.4.2 跟进记录规范

所有客户跟进均须在CRM系统中形成规范记录，记录的完整性和真实性是营销管理考核的核心依据。

**跟进记录填写规范**：

| 跟进维度 | 填写要求 | 示例 |
|----------|----------|----------|
| 跟进时间 | 填写实际跟进时间，不得虚假填写 | 2026-03-25 14:30 |
| 跟进方式 | 须与实际一致 | 上门拜访 |
| 跟进对象 | 填写实际接触人员姓名和职务 | 张总（董事长） |
| 跟进内容 | 详细记录沟通要点，不少于50字 | 就网络系统升级方案进行深度沟通，确认客户对现有网络架构的痛点：核心机房设备老化，稳定性不足；现有网络无法支持新业务系统扩展；网络安全防护需升级。客户明确表示上半年有预算开展该项目。 |
| 客户反馈 | 如实记录客户意见和态度 | 客户对我方技术方案认可度高，但希望了解具体报价和实施周期。 |
| 下一步行动 | 明确下一步跟进计划 | 下周提交详细方案和报价，约定4月初进行方案汇报。 |

**合规要求**：
- 跟进记录须在跟进完成后24小时内填写，逾期系统限制补录
- 跟进内容不得空洞敷衍（如仅填写"与客户沟通"而无实质内容）
- 跟进记录作为业务真实性核查的核心依据，虚假记录从重处理
- 涉及客户敏感商业信息的记录须严格保密

## E.2 营销数据分析工具操作规程

### E.2.1 数据看板使用规范

#### E.2.1.1 数据看板登录与界面介绍

营销数据分析看板是辅助管理层和营销人员掌握业务进展的核心工具，通过可视化图表呈现关键业绩指标。

**登录操作步骤**：

**步骤1：打开飞书工作台**
在飞书客户端中点击"工作台"，找到"营销数据看板"应用。

**步骤2：进入数据看板**
点击应用图标进入，默认展示个人业绩看板视图。

**步骤3：选择看板视图**
数据看板提供多种视图切换：

| 视图类型 | 适用角色 | 主要内容 |
|----------|----------|----------|
| 个人业绩看板 | 全体营销人员 | 本人客户数、商机金额、跟进记录、业绩完成率 |
| 团队业绩看板 | 销售经理/总监 | 团队整体业绩、各成员排名、目标完成进度 |
| 商机阶段看板 | 管理层 | 全公司商机分布、阶段转化率、重点商机跟踪 |
| 客户分析看板 | 管理层 | 客户行业分布、来源渠道分析、客户流失预警 |

**步骤4：时间维度筛选**
在看板顶部选择时间范围：今日/本周/本月/本季度/本年/自定义。

**注意事项**：
- 看板数据每小时自动刷新一次，如需实时数据点击刷新按钮手动刷新
- 看板数据以CRM系统为基础数据源，数据延迟不超过2小时
- 如发现看板数据与CRM数据存在明显差异，立即向数据管理员反馈

#### E.2.1.2 关键指标解读规范

| 指标名称 | 定义 | 解读标准 |
|----------|------|----------|
| 客户总量 | CRM系统中本人名下所有客户数量（含沉默客户） | — |
| 有效客户量 | 近90天内有跟进记录的客户数量 | 低于客户总量60%为预警 |
| 商机总额 | 本人名下所有处于初步接触至商务谈判阶段商机的预计金额总和 | — |
| 预计签单额 | 本季度预计签单的商机金额总和 | — |
| 实际签单额 | 本季度已实际签约的合同金额总和 | — |
| 业绩完成率 | 实际签单额÷季度目标×100% | 低于80%为预警 |
| 跟进覆盖率 | 有跟进记录的工作日÷总工作日×100% | 低于90%为预警 |
| 商机转化率 | 赢单商机数÷跟进商机总数×100% | 低于15%为预警 |

**合规要求**：
- 数据看板仅供内部管理使用，不得截图外传
- 各项指标数据须如实反映业务实际情况，不得为完成指标而虚假操作
- 指标预警提示仅为辅助参考，不得作为考核的唯一依据

### E.2.2 数据提取操作规程

#### E.2.2.1 客户数据导出

在需要进行客户信息统计、客户联系、营销活动等场景时，可从CRM系统导出客户数据。

**客户数据导出操作步骤**：

**步骤1：进入客户列表**
登录CRM系统，点击"客户管理"进入客户列表页。

**步骤2：设置筛选条件**
通过筛选器设置数据范围：

| 筛选字段 | 选项说明 |
|----------|----------|
| 客户类型 | 全部/潜在客户/意向客户/成交客户/沉默客户 |
| 创建时间 | 自定义日期范围 |
| 所属营销人员 | 全部/指定人员 |
| 所属行业 | 全部/指定行业 |
| 客户来源 | 全部/指定来源 |
| 客户等级 | 全部/A级/B级/C级 |

**步骤3：确认导出范围**
筛选完成后确认列表中的客户数量和范围是否符合导出需求。

**步骤4：执行导出**
点击右上角"导出"按钮，选择导出格式（Excel/CSV）。

**步骤5：填写导出申请**
系统弹出导出申请表单，填写：

| 申请字段 | 填写说明 |
|----------|----------|
| 导出用途 | 简述数据导出的业务用途 |
| 数据范围 | 自动显示当前筛选条件 |
| 涉及敏感信息 | 如客户手机号、邮箱等，需确认保密承诺 |

**步骤6：提交并等待处理**
提交申请后等待系统处理（一般不超过5分钟），完成后系统通知下载。

**注意事项**：
- 单次导出数据量不超过5000条，超过需分批导出
- 导出的客户手机号、邮箱等个人信息须严格保密
- 数据导出后须在本地做好安全防护（加密存储、阅后删除）
- 每月导出次数不超过20次

#### E.2.2.2 业绩报表导出

**业绩报表导出操作步骤**：

**步骤1：进入报表中心**
在CRM系统左侧菜单点击"报表中心"。

**步骤2：选择报表类型**
根据需求选择报表模板：

| 报表类型 | 内容说明 |
|----------|----------|
| 个人业绩报表 | 本人指定周期内的业绩数据汇总 |
| 团队业绩报表 | 指定团队指定周期内的业绩数据（需相应权限） |
| 商机阶段报表 | 各阶段商机数量和金额统计 |
| 跟进记录报表 | 指定周期内的跟进记录明细 |

**步骤3：设置报表参数**
设置时间范围（起止日期）、数据维度等参数。

**步骤4：生成报表**
点击"生成报表"，系统开始运算（数据量大时可能需要等待1-3分钟）。

**步骤5：预览并导出**
报表生成后支持在线预览，确认无误后点击"导出"下载。

**合规要求**：
- 业绩报表导出须注明用途，涉及绩效考核数据不得随意传播
- 团队业绩报表仅限团队负责人及以上层级导出
- 报表数据须与CRM原始数据一致，发现差异立即反馈

### E.2.3 报表生成操作规程

#### E.2.3.1 定制化报表创建

如标准报表模板无法满足需求，可创建定制化报表。

**定制化报表创建操作步骤**：

**步骤1：进入报表设计器**
在报表中心点击"+新建报表"，进入报表设计器。

**步骤2：选择数据源**
选择报表数据来源（客户数据/商机数据/跟进数据/业绩数据）。

**步骤3：选择数据字段**
从可用字段列表中选择需要展示的字段，拖入报表设计区：

| 字段分类 | 可选字段 |
|----------|----------|
| 客户字段 | 客户名称/类型/行业/规模/来源/创建时间/最近跟进时间 |
| 商机字段 | 商机名称/阶段/预计金额/预计签单日期/胜算评估 |
| 跟进字段 | 跟进时间/方式/对象/内容摘要/跟进人员 |
| 业绩字段 | 签单金额/合同编号/签单日期/毛利率 |

**步骤4：设置筛选条件**
为报表设置默认筛选条件，如固定只看本季度数据等。

**步骤5：设置图表类型**
为报表数据选择展示图表类型：

| 图表类型 | 适用场景 |
|----------|----------|
| 数据表格 | 明细数据展示，需要精确数值 |
| 柱状图 | 不同类别数据对比（如各行业客户数量对比） |
| 折线图 | 趋势变化展示（如月度签单额趋势） |
| 饼图 | 占比分析（如客户类型占比） |

**步骤6：保存并命名报表**
填写报表名称（如"2026年Q1客户行业分布报表"），点击保存。

**步骤7：设置定期推送**（可选）
如需定期生成同一报表，可设置自动推送计划（每日/每周/每月定时推送至指定邮箱或飞书群）。

**注意事项**：
- 定制化报表创建权限仅向管理层开放，一线营销人员如需定制报表可向主管申请代为创建
- 报表命名须规范，体现报表内容实质
- 定期推送报表须在报表内容稳定后再设置自动推送

## E.3 营销自动化工具操作规程

### E.3.1 线索分配深度操作规程

#### E.3.1.1 线索录入与来源管理

线索是营销漏斗的入口，所有通过各渠道获取的潜在客户信息均须录入线索管理系统。

**线索录入操作步骤**：

**步骤1：进入线索管理模块**
在CRM系统中点击"线索管理"进入线索管理页面。

**步骤2：点击新建线索**
点击右上角"+新建线索"按钮。

**步骤3：填写线索基本信息**

| 字段名称 | 填写要求 | 示例 |
|----------|----------|------|
| 客户名称 | 必填，目标客户的单位全称 | 青岛某某工贸有限公司 |
| 联系人姓名 | 必填，真实姓名 | 李明 |
| 手机号 | 必填，11位有效手机号 | 138XXXXXXXX |
| 邮箱 | 选填，验证格式有效性 | liming@xxx.com |
| 所属行业 | 从下拉列表选择 | 制造业 |
| 所在地区 | 精确到地级市 | 山东省青岛市 |
| 需求类型 | 多选：网络系统集成/智能安防/IT运维/软件服务/综合解决方案 | 网络系统集成,智能安防 |
| 需求描述 | 简述客户需求概况（不少于30字） | 客户计划上半年对办公网络进行升级改造，需更换核心交换机和无线覆盖系统，同时增加网络监控点位约20个。 |
| 线索来源 | 单选：官网表单/展会收集/电话营销/地推陌拜/老客推荐/平台引流/其他 | 官网表单 |

**步骤4：填写来源详情**
根据线索来源填写详细信息：

| 来源类型 | 需填写的详情字段 |
|----------|------------------|
| 官网表单 | 表单提交时间、表单类型、咨询页面URL |
| 展会收集 | 展会名称、收集时间、收集人员 |
| 电话营销 | 拨打时间、通话时长、意向确认 |
| 老客推荐 | 推荐人姓名、推荐人联系方式、被推荐客户名称 |

**步骤5：保存线索**
确认信息无误后保存。系统自动生成线索编号并进入待分配状态。

**注意事项**：
- 线索信息须真实有效，禁止录入虚假客户信息
- 同一客户不得重复录入线索，如已在CRM中有客户档案则新建商机而非线索
- 线索来源须准确填写，以便后续分析各渠道线索质量
- 录入完成后须在24小时内完成首次联系并记录结果

#### E.3.1.2 线索分配规则与流程

**线索自动分配规则**：

| 分配维度 | 分配规则 |
|----------|----------|
| 区域分配 | 按客户所在地区分配至对应区域的营销人员 |
| 行业分配 | 按客户所属行业分配至对应行业的营销人员 |
| 轮询分配 | 各渠道线索按顺序轮流分配给空闲营销人员 |
| 均衡分配 | 确保各营销人员待跟进线索数量不超过上限（10条） |

**线索手动分配操作步骤**：

**步骤1：进入线索分配页面**
系统管理员或销售经理点击"线索管理"中的"分配管理"模块。

**步骤2：选择待分配线索**
勾选需要分配的线索（支持批量选择，单次最多50条）。

**步骤3：选择接收人**
从营销人员列表中选择接收线索的人员。可按姓名、部门进行搜索。

**步骤4：填写分配备注**
在分配备注中说明分配原因和跟进要求。

**步骤5：确认分配**
点击"确认分配"按钮。系统自动向接收人发送分配通知。

**合规要求**：
- 线索分配须遵循公平原则，不得选择性分配或囤积线索
- 线索分配记录永久留存，可追溯至每个线索的分配历史
- 营销人员不得拒绝正常分配的线索，如有特殊情况须书面说明原因
- 连续3次无故拒绝分配线索纳入绩效考核扣分项

#### E.3.1.3 线索转化与跟进时限

**线索转化操作步骤**：

**步骤1：完成首次有效跟进**
在联系客户后，如客户有明确需求意向，进入线索详情页。

**步骤2：评估转化条件**
确认满足以下转化条件：

| 转化条件 | 判断标准 |
|----------|----------|
| 客户确认需求 | 客户明确表示有相关项目需求 |
| 联系对象正确 | 联系到了解项目并能做决策或推动的人 |
| 信息完整度 | 客户名称、联系人、手机号、需求描述均完整 |
| 跟进记录 | 已完成首次有效跟进并记录 |

**步骤3：申请转化为客户**
点击线索详情页的"转为客户"按钮，填写转化信息：

| 转化字段 | 填写说明 |
|----------|----------|
| 转化结果 | 单选：成功转化/未能转化 |
| 未转化原因 | 如未转化，选择原因：客户无需求/联系不上/信息错误/其他 |
| 跟进总结 | 总结本次线索跟进的得失（不少于50字） |

**步骤4：系统自动处理**
成功转化：系统自动在CRM中创建客户档案，并继承线索中的客户信息。
未能转化：线索标记为"未能转化"，进入线索库备查。

**跟进时限规定**：

| 线索类型 | 首次跟进时限 | 跟进间隔要求 |
|----------|--------------|--------------|
| 高质量线索（来源：官网表单/老客推荐） | 4小时内 | 每3天跟进一次 |
| 中质量线索（来源：展会/平台引流） | 24小时内 | 每7天跟进一次 |
| 普通线索（来源：电话营销/地推陌拜） | 48小时内 | 每14天跟进一次 |

**注意事项**：
- 超时未跟进系统自动预警并通知主管
- 连续3次联系无果的线索可申请暂时搁置，但须说明原因并经主管审批
- 被搁置线索如客户主动再次联系，系统自动重新激活并提醒原跟进人

### E.3.2 跟进提醒工具操作规程

#### E.3.2.1 跟进任务创建与提醒设置

跟进提醒是确保销售跟进不掉节的关键工具，营销人员须善用提醒功能管理客户跟进节奏。

**跟进任务创建操作步骤**：

**步骤1：进入任务管理**
在CRM系统左侧菜单点击"任务管理"，进入任务列表页。

**步骤2：创建新任务**
点击"+新建任务"按钮。

**步骤3：填写任务信息**

| 任务字段 | 填写要求 |
|----------|----------|
| 任务标题 | 必填，简明扼要，如"某某科技方案报价跟进" |
| 关联客户 | 从客户列表选择关联客户 |
| 关联商机 | 如有，关联对应商机 |
| 任务类型 | 单选：客户跟进/商机推进/会议预约/合同签署/其他 |
| 截止时间 | 必填，设置任务截止日期和时间 |
| 优先级 | 单选：高/中/低 |
| 提醒时间 | 设置提醒时间（建议提前1小时至1天） |
| 任务描述 | 详细说明任务内容和具体要求 |
| 任务执行人 | 默认当前用户，可指派给他人 |

**步骤4：保存任务**
确认信息无误后保存。系统自动在设定时间发送提醒通知。

**步骤5：执行任务**
按提醒执行任务，完成后在任务详情页点击"完成"并填写执行结果。

**注意事项**：
- 任务设置须具体可执行，避免设置模糊任务（如"跟进客户"而无可执行内容）
- 每日任务数量建议控制在10个以内，优先级合理分配
- 已过期未完成的任务须在24小时内处理（完成或延期）

#### E.3.2.2 自动化提醒规则配置

**系统级自动化提醒规则**：

**合规要求**：
- 自定义提醒规则须报备主管，确保提醒内容符合业务规范
- 提醒频率须合理，不得设置过于频繁的提醒（间隔少于2小时视为过于频繁）

### E.3.3 自动化工作序列操作规程

#### E.3.3.1 自动化序列创建与配置

自动化工作序列是提高营销效率的核心工具，通过预设的工作流程自动执行重复性任务。

**自动化序列创建操作步骤**：

**步骤1：进入自动化中心**
在CRM系统左侧菜单点击"自动化中心"。

**步骤2：创建新序列**
点击"+新建自动化序列"。

**步骤3：设置序列基本信息**

| 字段名称 | 填写要求 |
|----------|----------|
| 序列名称 | 必填，体现序列用途，如"新线索7日培育序列" |
| 序列类型 | 单选：线索培育/客户激活/商机推进/活动通知/其他 |
| 适用对象 | 设置序列适用的客户/线索条件 |
| 序列有效期 | 设置序列的有效起止时间 |

**步骤4：设计序列节点**
在序列画布中拖拽添加节点：

| 节点类型 | 功能说明 |
|----------|----------|
| 触发节点 | 设定序列的启动条件（如：新线索创建时自动启动） |
| 延时节点 | 设置等待时间（如：触发后等待3天） |
| 邮件节点 | 自动发送邮件（如：发送产品介绍邮件） |
| 短信节点 | 自动发送短信（如：提醒客户查看报价） |
| 任务节点 | 自动创建跟进任务（如：创建上门拜访任务） |
| 条件节点 | 根据客户行为分流（如：打开邮件则进入A分支，未打开则进入B分支） |
| 结束节点 | 标记序列结束，可设置结局状态 |

**步骤5：配置节点参数**
为每个节点设置具体参数（如邮件主题、短信内容、任务描述等）。

**步骤6：保存并激活**
确认序列配置无误后，点击"保存"并开启"激活"开关。

**注意事项**：
- 自动化序列会影响大量客户，修改前须充分评估影响范围
- 同一客户同一时间只能在一个激活序列中
- 序列激活后系统自动执行，中途停止需谨慎操作

## E.4 营销物料管理系统操作规程

### E.4.1 宣传物料管理规范

#### E.4.1.1 宣传物料分类与编码

宣传物料是营销业务开展的重要支撑物资，须进行规范分类管理。

**宣传物料分类表**：

| 物料大类 | 物料细类 | 示例物品 |
|----------|----------|----------|
| 公司宣传册 | 产品宣传册/解决方案手册/成功案例集 | 《道简网络智能安防解决方案》《道简网络成功案例集》 |
| 演示物料 | 产品样品/演示设备/演示模型 | 网络设备样品、安防摄像头样品 |
| 宣传品 | 名片/PPT模板/宣传单页/产品彩页 | 标准名片、年度汇报PPT模板 |
| 展会物料 | 展架/横幅/背景板/宣传视频U盘 | 3x3米标准展架、1分钟公司介绍视频 |
| 定制礼品 | 办公文具/电子礼品/公司定制礼品 | U盘、充电宝，品牌记事本 |

**物料编码规则**：

| 编码结构 | 说明 | 示例 |
|----------|------|------|
| 大类码-小类码-序号 | 大类用字母，小类用字母，序号用数字 | CP-XN-001（产品宣传册-解决方案-001号） |
| 命名格式 | 物料名称+版本+印刷/采购日期 | 道简网络智能安防彩页V2.0-202603 |

**注意事项**：
- 物料编码须唯一，禁止重复编码
- 物料名称须准确反映物料内容
- 版本更新后旧版本物料须及时下架销毁

#### E.4.1.2 物料领用流程

**物料领用操作步骤**：

**步骤1：进入物料管理系统**
在飞书工作台中找到营销物料管理系统应用，点击进入。

**步骤2：查看物料库存**
在物料目录中浏览或在搜索框中搜索所需物料，查看当前库存情况。

**步骤3：提交领用申请**
点击物料条目右侧的领用按钮，填写领用申请。

**步骤4：填写领用信息**

| 申请字段 | 填写说明 |
|----------|----------|
| 领用数量 | 填写需要领用的数量 |
| 领用用途 | 简述物料使用场景和目的 |
| 预计使用日期 | 填写物料预计使用的日期 |
| 预计归还日期 | 如是借用类物料，填写预计归还日期 |

**步骤5：提交申请并等待审批**
确认领用信息无误后提交。系统将申请推送至物料管理员进行审批（一般1个工作日内完成）。

**步骤6：领取物料**
审批通过后，前往物料管理员处领取物料，并在物料领用登记表上签字确认。

**注意事项**：
- 物料领用须提前申请，紧急情况可先电话联系物料管理员说明情况，事后补办手续
- 名片，宣传册等常规物料单次领用不超过200份，超出需说明原因
- 借用类物料须按期归还，逾期未还系统自动提醒
- 物料领用后如未使用须归还，不得私藏私用

#### E.4.1.3 物料库存盘点

**库存盘点操作步骤**：

**步骤1：进入物料管理模块**
在营销物料管理系统中点击库存管理模块。

**步骤2：发起盘点任务**
点击新建盘点任务，填写盘点信息：

| 盘点字段 | 填写说明 |
|----------|----------|
| 盘点类型 | 单选：全面盘点/抽样盘点 |
| 盘点范围 | 选择需要盘点的物料类别 |
| 盘点负责人 | 指定本次盘点的负责人 |
| 计划完成日期 | 指定盘点任务的计划完成日期 |

**步骤3：执行实地盘点**
盘点人员前往物料存放地点，逐项清点物料数量，与系统记录进行核对。

**步骤4：记录盘点结果**
在盘点任务中填写实际盘点数量，如发现差异（盈亏）须注明差异原因。

**步骤5：提交盘点报告**
盘点完成后提交盘点报告，系统自动更新库存数据。

**注意事项**：
- 库存盘点每月至少进行一次全面盘点
- 发现物料过期、损坏须及时填写报废申请
- 盘点结果与系统数据差异超过5%须提交书面说明

### E.4.2 样品管理规范

#### E.4.2.1 样品分类与编号

**样品分类表**：

| 样品类别 | 示例物品 | 管理要求 |
|----------|----------|----------|
| 网络设备样品 | 交换机、路由器、无线AP、防火墙 | 须登记设备型号、序列号 |
| 安防设备样品 | 摄像头、录像机、门禁设备 | 须登记设备型号、MAC地址 |
| 线材辅材样品 | 网线，光纤、水晶头、线管 | 按数量管理，精度要求较低 |
| 软件产品样品 | 软件安装介质、授权密钥 | 须登记授权数量和有效期 |

**样品编号规则**：

| 编号结构 | 说明 | 示例 |
|----------|------|------|
| YP-设备类别码-序号 | YP代表样品，设备类别用字母 | YP-NET-001（网络设备样品第1号） |

#### E.4.2.2 样品借用流程

**样品借用操作步骤**：

**步骤1：提交借用申请**
在物料管理系统中点击样品管理，点击借用申请，填写借用信息：

| 借用字段 | 填写说明 |
|----------|----------|
| 样品名称 | 从样品列表选择 |
| 借用数量 | 填写需要借用的数量 |
| 借用用途 | 简述样品使用目的（如客户演示、项目测试等） |
| 借用期限 | 填写预计归还日期（一般不超过30天） |
| 借用人 | 自动填充当前用户 |

**步骤2：等待审批**
借用申请提交后由样品管理员进行审批。

**步骤3：领取样品**
审批通过后，前往样品管理员处领取样品。

**步骤4：使用与归还**
按借用用途使用样品，使用完毕后及时归还。

**步骤5：归还确认**
归还时由样品管理员检查样品状态，在系统中确认归还并更新样品状态。

**注意事项**：
- 样品借用须经样品管理员审批，高价值样品须经部门主管额外审批
- 借用期间须妥善保管样品，如有人为损坏须照价赔偿
- 借用期满如需续借须提前申请续借审批
- 样品归还时须保持清洁、完好状态

#### E.4.2.3 样品报废与处置

**样品报废条件**：
- 设备损坏无法修复
- 设备老化、技术淘汰
- 软件样品授权过期且不可续期
- 存储样品超过保质期

**样品报废操作步骤**：

**步骤1：提交报废申请**
在样品管理模块中点击报废申请，填写报废原因。

**步骤2：技术鉴定**
由技术部门对待报废样品进行技术鉴定，确认无法继续使用。

**步骤3：审批处理**
报废申请经样品管理员和部门主管审批后执行。

**步骤4：系统处理**
报废完成后在系统中更新样品状态为已报废，并记录报废日期和处置方式。

**合规要求**：
- 样品报废须严格执行审批流程，不得私自处置
- 报废样品中含有客户信息的须进行数据清除
- 报废样品的处置须符合公司资产管理规定

## E.5 营销工具安全与权限管理规范

### E.5.1 账号安全规范

#### E.5.1.1 账号注册与开通

**账号开通申请流程**：

| 序号 | 操作步骤 | 负责人 |
|------|----------|--------|
| 1 | 新员工提交账号开通申请，注明需要开通的系统模块 | 员工本人 |
| 2 | 部门主管审批账号申请，确认申请权限的合理性 | 部门主管 |
| 3 | IT管理员根据审批结果开通相应权限 | IT管理员 |
| 4 | 账号开通后向申请人发送通知，告知初始密码 | IT管理员 |
| 5 | 申请人首次登录后立即修改初始密码 | 员工本人 |

**账号开通类型说明**：

| 账号类型 | 适用人员 | 默认权限范围 |
|----------|----------|-------------|
| 标准营销账号 | 一线营销人员 | 客户管理、商机管理、跟进记录、数据查看 |
| 团队管理账号 | 销售经理/总监 | 标准权限+团队管理、数据统计、报表导出 |
| 系统管理账号 | IT管理员/销售总监 | 全功能权限+系统配置、权限管理、数据删除 |

**注意事项**：
- 账号开通须遵循最小权限原则，按实际工作需要申请权限
- 临时账号须注明使用期限，到期自动失效
- 一人多账号须向IT部门报备

#### E.5.1.2 密码安全规范

**密码设置要求**：

| 要求项 | 具体要求 |
|--------|----------|
| 密码长度 | 最少8位 |
| 字符要求 | 必须包含大小写字母和数字 |
| 特殊字符 | 建议包含@、#、$等特殊字符 |
| 禁止项 | 禁止使用简单密码（如12345678、password等） |
| 禁止项 | 禁止使用与本人信息相关的密码（如生日、手机号等） |

**密码更换周期**：

| 账号类型 | 建议更换周期 |
|----------|-------------|
| 普通营销账号 | 90天内更换一次 |
| 团队管理账号 | 60天内更换一次 |
| 系统管理账号 | 30天内更换一次 |

**密码泄露应急处理**：

| 步骤 | 操作内容 |
|------|----------|
| 1 | 立即更改密码 |
| 2 | 检查账号下是否存在异常操作记录 |
| 3 | 如发现异常操作，立即向IT部门报告 |
| 4 | IT部门排查原因并采取进一步措施 |
| 5 | 必要时冻结账号进行彻查 |

**合规要求**：
- 密码须严格保密，不得向他人透露
- 不得在非公司设备上保存密码
- 发现密码泄露须立即报告，不得隐瞒

#### E.5.1.3 账号权限管理

**权限分类说明**：

| 权限级别 | 权限内容 | 适用对象 |
|----------|----------|----------|
| 查看权限 | 仅可查看自己创建的数据 | 普通营销人员 |
| 编辑权限 | 可查看和编辑本人及分配给自己的数据 | 营销人员 |
| 管理权限 | 可管理本团队数据，查看团队成员数据 | 团队负责人 |
| 全局权限 | 可管理全部数据，系统配置权限 | 系统管理员 |

**权限变更申请流程**：

**步骤1：提交权限变更申请**
员工向IT管理员提交权限变更申请，说明需要增加或减少的权限及原因。

**步骤2：部门主管审批**
部门主管审批权限变更申请的合理性。

**步骤3：IT管理员执行**
IT管理员根据审批结果调整账号权限。

**步骤4：变更确认**
权限调整完成后向申请人发送确认通知。

**注意事项**：
- 权限变更须经部门主管审批，不得私下向IT管理员要求权限调整
- 权限调整记录永久留存
- 离职或调岗时须及时收回相关权限

### E.5.2 数据安全管理

#### E.5.2.1 数据保密等级分类

**数据保密等级表**：

| 保密等级 | 定义 | 示例数据 | 保护要求 |
|----------|------|----------|----------|
| 绝密 | 公司核心商业机密，泄露会造成重大损失 | 战略规划、核心产品价格体系、重大收购计划 | 仅限必要知悉人员，访问须审批，禁止外传 |
| 机密 | 重要商业信息，泄露会造成较大损失 | 客户详细资料、合同金额、未公开财务数据 | 限相关业务人员，访问须记录，禁止外传 |
| 秘密 | 一般内部信息，泄露会造成一定影响 | 内部工作流程、非公开业务数据 | 限公司内部人员，遵守公司规定 |
| 内部 | 仅限公司内部使用 | 公司公告、一般业务数据 | 不得向外部人员透露 |
| 公开 | 可对外公开的信息 | 公司官网内容、公开宣传资料 | 无限制 |

#### E.5.2.2 数据访问与导出规范

**数据访问规范**：

| 数据类别 | 访问权限说明 |
|----------|-------------|
| 本人创建的数据 | 创建者可查看、编辑、删除 |
| 分配给本人的数据 | 接收人可查看、编辑、删除 |
| 团队成员数据 | 团队负责人可查看，不得删除他人数据 |
| 全部数据 | 仅系统管理员可访问 |

**数据导出规范**：

| 数据类型 | 导出要求 |
|----------|----------|
| 客户联系方式（手机号、邮箱等） | 须在导出申请中注明用途，经部门主管审批 |
| 业绩数据 | 仅限本人或本团队数据，跨团队数据须审批 |
| 商机信息 | 涉及商业秘密，导出须经审批 |
| 系统全部数据 | 仅限系统管理员，并须记录用途 |

**合规要求**：
- 数据导出须严格遵守公司数据安全规定
- 导出的数据仅可用于申请时填写的用途，不得挪作他用
- 导出数据须采取安全防护措施（加密存储、阅后删除等）
- 发现数据泄露须立即报告并采取补救措施

#### E.5.2.3 数据备份与恢复

**数据备份策略**：

| 备份类型 | 备份频率 | 保存周期 | 责任人 |
|----------|----------|----------|--------|
| 全量备份 | 每周一次 | 12个月 | IT管理员 |
| 增量备份 | 每日一次 | 3个月 | IT管理员 |
| 实时备份 | 关键操作实时同步 | 永久 | 系统自动 |

**数据恢复流程**：

**步骤1：提交数据恢复申请**
用户向IT部门提交数据恢复申请，说明需要恢复的数据类型和时间范围。

**步骤2：IT部门评估**
IT部门评估恢复可行性及对其他数据的影响。

**步骤3：执行恢复**
在备份数据中定位需要恢复的数据，执行恢复操作。

**步骤4：确认恢复结果**
恢复完成后由申请人确认数据完整性和正确性。

**注意事项**：
- 数据恢复可能影响最新数据，须谨慎操作
- 涉及其他用户的数据恢复须通知相关人员
- 数据恢复记录须保存备查

### E.5.3 合规审计管理

#### E.5.3.1 审计内容与范围

**常规审计内容**：

| 审计类别 | 审计内容 | 审计周期 |
|----------|----------|----------|
| 登录审计 | 登录时间、登录IP、登录是否成功 | 实时记录，每月抽查 |
| 操作审计 | 数据创建、修改、删除操作记录 | 实时记录，每月抽查 |
| 权限审计 | 权限变更记录、异常权限使用 | 每月全面审计 |
| 数据导出审计 | 数据导出记录、导出内容、导出用途 | 实时记录，每周抽查 |
| 批量操作审计 | 批量删除、批量转移等操作的执行记录 | 实时记录，每次必查 |

#### E.5.3.2 违规行为界定

**严重违规行为（红色）**：
- 盗用他人账号或密码
- 故意删除、篡改他人业务数据
- 未经授权导出大量客户数据并外传
- 恶意破坏系统数据或系统运行
- 为谋取私利伪造业务数据

**较严重违规行为（橙色）**：
- 借用他人账号操作
- 未经批准擅自导出客户联系方式
- 违反数据保密规定导致信息泄露
- 恶意删除本人业务数据影响工作

**一般违规行为（黄色）**：
- 长期使用简单密码（超过规定周期1倍以上）
- 未按规定及时更新客户信息
- 未经批准进行大批量数据操作
- 离开工位未锁定系统界面

#### E.5.3.3 违规处理与问责

**违规处理流程**：

| 处理阶段 | 操作内容 | 责任人 |
|----------|----------|--------|
| 发现与报告 | 发现违规行为，向IT部门和主管部门报告 | 发现人 |
| 调查核实 | IT部门调查取证，确认违规事实 | IT部门 |
| 通知当事人 | 向违规者告知违规事实和拟处理意见 | 主管部门 |
| 当事人申诉 | 违规者可在5个工作日内提出申诉 | 当事人 |
| 做出处理决定 | 根据违规情节轻重做出处理决定 | 主管部门 |
| 执行处理 | 执行相应处理措施并记录 | IT部门 |
| 整改跟踪 | 跟进整改情况，确保问题解决 | 主管部门 |

**违规处理标准**：

| 违规等级 | 处理措施 |
|----------|----------|
| 红色违规 | 冻结账号、解除劳动关系、依法追究法律责任 |
| 橙色违规 | 冻结账号1-3个月、绩效奖金扣减、全公司通报批评 |
| 黄色违规 | 口头警告或书面警告、要求限期整改、扣减部分绩效 |

**合规要求**：
- 所有违规处理须有书面记录并存档
- 处理决定须告知当事人并给予申诉机会
- 涉及法律责任的须移交司法机关处理
- 定期进行合规培训，提高全员安全意识
