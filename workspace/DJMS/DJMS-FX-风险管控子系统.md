## 1. 风险管控子系统概述

### 1.1 定位与职责

风险管控子系统是道简网络运行系统（DJRS）管控层的**风险屏障**，承担着识别、评估、监控和应对企业内外部风险的关键职能，是 DJMS 五大子系统的安全保障中心。

**核心职责**：
- 建立道简网络全面风险管理体系，覆盖战略/财务/运营/合规/项目/网络安全等维度
- 识别和评估企业面临的各类风险（风险登记册维护）
- 建立风险预警机制，对接 DJMS-JS 计划预算子系统的预算预警
- 推动重大风险的应对策略制定与执行跟踪
- 管理危机事件与应急响应
- 对接 DJMS-SJ 审计管控子系统，为审计提供风险信息

**管控边界**：
- 企业全面风险管理（识别/评估/应对/监控）
- 安全与合规管理（网络安全/数据安全/合规）
- 项目风险管理（项目交付风险/客户风险）
- 危机管理与应急响应

### 1.2 模块架构

风险管控子系统包含以下九大核心模块：

## 3. 各模块详细流程

### 3.1 战略风险管控（3.1.1）

#### 3.1.1 模块职责

- 监测道简网络战略执行偏差风险
- 评估战略环境变化带来的风险（市场/政策/技术）
- 跟踪 DJMS-ZL 战略管控子系统的战略回顾结果
- 管理重大战略决策的风险评估流程
- 建立战略风险预警指标体系
- 对接 DJMS-ZL 的战略回顾机制

#### 3.1.2 一级流程：战略风险管控

```plaintext
【战略风险管控一级流程】

战略风险识别
    │
    ├─ 战略环境变化监测（市场/政策/技术）
    ├─ 战略执行偏差监测（KPI达成）
    └─ 战略决策风险评估（新战略举措）
    │
    ↓
战略风险评估
    │
    ├─ 风险影响程度（重大/较大/一般）
    ├─ 风险发生概率（高/中/低）
    └─ 风险等级（红/橙/黄/蓝）
    │
    ↓
战略风险应对
    │
    ├─ 规避（调整战略方向）
    ├─ 降低（加强执行力度）
    └─ 承受（接受并监控）
    │
    ↓
战略风险监控
    │
    ├─ 建立战略风险指标
    └─ 季度风险评估更新
    │
    ↓
对接DJMS-ZL战略回顾
```

#### 3.1.3 战略风险评估标准

| 战略风险类型 | 风险触发条件 | 影响程度 | 概率评估 | 风险等级 | 应对策略 |
|--------------|--------------|----------|----------|----------|----------|
| 市场萎缩风险 | 目标市场规模下降>20% | 重大 | 中 | 橙色 | 业务转型/新市场拓展 |
| 政策不利风险 | 核心政策重大调整 | 重大 | 低 | 橙色 | 业务调整/合规应对 |
| 技术替代风险 | 核心技术被替代 | 重大 | 中 | 橙色 | 技术升级/转型 |
| 战略执行风险 | 连续2季度KPI达成率<70% | 较大 | 中 | 黄色 | 战略纠偏/资源重新配置 |
| 竞争格局恶化 | 主要竞争对手市场份额提升>15% | 较大 | 中 | 黄色 | 竞争策略调整 |
| 核心人才流失 | 核心技术/销售人才离职 | 重大 | 低 | 黄色 | 人才备份/激励机制 |

### 3.2 市场风险管控（3.1.2）

#### 3.2.1 模块职责

- 监测企业数字化、智能安防、系统集成、IT运维服务市场的外部风险
- 跟踪政府采购预算变化与政企IT投资趋势
- 分析竞争对手动态带来的市场风险
- 评估新业务/新区域拓展的市场风险
- 管理客户集中度风险
- 对接 DJMS-ZL 市场分析与竞争分析模块

#### 3.2.2 一级流程：市场风险管控

```plaintext
【市场风险管控一级流程】

市场风险信息收集
    │
    ├─ 政策信息（政府IT预算/信创政策）
    ├─ 行业数据（市场增速/细分领域变化）
    ├─ 竞争对手动态（价格战/新进入者）
    └─ 客户需求变化（需求萎缩/新需求）
    │
    ↓
市场风险识别
    │
    ├─ 行业周期性风险
    ├─ 客户集中度风险
    ├─ 竞争加剧风险
    └─ 需求变化风险
    │
    ↓
风险评估与定级
    ↓
制定应对策略
    │
    ├─ 客户结构调整（降低集中度）
    ├─ 业务多元化（降低行业依赖）
    └─ 竞争优势强化（差异化能力）
    │
    ↓
市场风险监控
```

#### 3.2.3 市场风险评估矩阵

| 市场风险类型 | 识别标准 | 评估维度 | 触发阈值 | 应对措施 |
|--------------|----------|----------|----------|----------|
| 行业萎缩风险 | 目标市场增速连续2年<5% | 影响程度 | 重大 | 新行业拓展 |
| 客户集中风险 | 单一客户收入占比>30% | 集中度 | 较大 | 客户分散 |
| 竞争加剧风险 | 主要对手报价低于我方15% | 价格影响 | 较大 | 差异化竞争 |
| 政采预算风险 | 政府IT预算削减>20% | 收入影响 | 重大 | 商业市场拓展 |
| 技术变革风险 | 主流技术路线重大变化 | 能力影响 | 中等 | 技术预研投入 |
| 供应商依赖风险 | 单一代割合作品占比>50% | 成本/交付 | 中等 | 备选供应商引入 |

#### 3.3.2 一级流程：财务风险管控

```plaintext

    │

    ├─ 预警指标监控

```

#### 3.3.3 财务风险预警指标

```plaintext
流动性风险预警：
    ├─ 现金比率 < 0.5 → 黄灯预警
    ├─ 现金比率 < 0.3 → 红灯预警
    ├─ 经营活动现金流连续3月为负 → 红灯预警
    └─ 银行授信使用率 >80% → 黄灯预警

应收账款风险预警：
    ├─ 逾期率 >15% → 黄灯预警
    ├─ 逾期率 >25% → 红灯预警
    ├─ 单笔大额逾期（>50万逾期超60天）→ 立即红灯
    └─ 客户信用评级下调 → 立即评估

成本风险预警：
    ├─ 项目成本超预算 >10% → 黄灯预警
    ├─ 项目成本超预算 >20% → 红灯预警
    └─ 整体成本增速超过收入增速5%以上 → 黄灯预警

税务风险预警：
    ├─ 税负率异常（低于行业均值30%以上）→ 黄灯预警
    └─ 税务稽查通知 → 红灯预警
```

| 财务风险指标 | 安全 | 预警（黄） | 危险（红） | 监控频率 |
|--------------|------|------------|------------|----------|
| 现金比率 | >0.5 | 0.3-0.5 | <0.3 | 月度 |
| 流动比率 | >1.5 | 1.0-1.5 | <1.0 | 月度 |
| 应收账款周转天数 | <60天 | 60-90天 | >90天 | 月度 |
| 逾期应收账款率 | <10% | 10-20% | >20% | 月度 |
| 资产负债率 | <60% | 60-75% | >75% | 季度 |
| 毛利率波动 | <3% | 3-5% | >5% | 季度 |

#### 3.6.3 项目风险识别与应对矩阵

```plaintext

    ├─ 进度滞后 → 赶工计划/资源追加

    ├─ 质量缺陷 → 质量检查/整改

    └─ 验收款拒付 → 法务介入/协商解决

项目运维阶段风险：
    ├─ 服务质量问题 → 服务标准提升/团队培训
    ├─ 客户不满意 → 客户关怀/补救措施
    └─ 续约困难 → 续约谈判/服务优化
```

#### 3.8.3 风险预警级别与升级机制

```plaintext
蓝色预警（一般风险）：
    ├─ 触发条件：单一指标轻微异常
    ├─ 通知范围：风险专员+部门负责人
    ├─ 响应时限：24小时内
    └─ 处置方式：部门自行处理

黄色预警（较大风险）：
    ├─ 触发条件：指标持续异常或多个指标异常
    ├─ 通知范围：风险负责人+部门负责人+财务总监
    ├─ 响应时限：8小时内
    └─ 处置方式：专项整改，3天内提交整改报告

橙色预警（重大风险）：
    ├─ 触发条件：重大指标严重偏离或突发事件
    ├─ 通知范围：风险负责人+总经理+董事会
    ├─ 响应时限：立即响应（1小时内）
    └─ 处置方式：成立专项小组，制定应急方案

红色预警（特别重大风险）：
    ├─ 触发条件：可能导致重大损失或经营中断
    ├─ 通知范围：全员管理层+董事会
    ├─ 响应时限：立即响应（15分钟内）
    └─ 处置方式：启动危机管理应急响应
```

| 预警级别 | 风险性质 | 指标偏差 | 潜在损失 | 升级时限 |

| 蓝色 | 一般 | <10% | <5万 | 24小时 |
| 黄色 | 较大 | 10-20% | 5-20万 | 8小时 |
| 橙色 | 重大 | 20-40% | 20-100万 | 1小时 |
| 红色 | 特别重大 | >40% | >100万 | 15分钟 |

### 4.2 持续优化机制

1. **年度风险管理评审**：每年末进行年度风险管理评审，更新风险登记册，评估风险管理体系有效性
2. **季度风险例会**：每季度召开风险例会，跟踪重大风险整改情况，识别新增风险
3. **月度风险报告**：每月编制风险监控月报，向管理层报告风险状况
4. **专项风险评估**：针对重大决策（新业务/重大项目/重大投资）开展事前风险评估
5. **危机演练**：每年至少组织一次危机演练，验证应急预案的有效性
