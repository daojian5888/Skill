# swimlane-diagram 中文介绍

## 技能定位

`swimlane-diagram` 是一个专门用于生成标准泳道图的 Codex 技能。

它不是简单把文字流程转成 Mermaid 代码，也不是只给出一份“能看但不正式”的流程草图。

它的核心目标非常明确：

- 把业务流程、SOP、制度流程、审批流程、交付流程等内容
- 转成真正可展示、可汇报、可沉淀的泳道图成品
- 默认输出高质量 `SVG` 图形
- 优先适配企业管理、组织协同和经营流程场景

如果一句话概括，它解决的是：

**如何把“文字里的流程”变成“标准、清晰、能直接拿来用的正式泳道图”。**

## 为什么需要这个技能

很多团队在做流程梳理时，都有同样的问题：

- 流程说明只有文字，没有图
- 不同岗位对流程理解不一致
- 图虽然画出来了，但责任边界不清楚
- 流程节点很多，线条杂乱，难以阅读
- Mermaid 图太像技术草稿，不像正式业务文档
- 普通流程图没有体现跨部门协同和交接关系
- 图的排版、节点、箭头、颜色风格不统一，专业感很弱

这些问题在企业管理场景里尤其突出。

因为企业管理类流程，通常都有以下特点：

- 角色多
- 交接多
- 审批多
- 例外情况多
- 需要体现责任边界
- 需要同时兼顾“能执行”和“能展示”

`swimlane-diagram` 正是为这种场景设计的。

它不是把图“画出来”就结束，而是把图的结构、版式和质量要求，一开始就固化成规则。

## 核心优势

### 1. 默认输出 SVG 成品图，而不是默认输出 Mermaid

这是这个技能最核心的优势。

很多流程图技能最后输出的是：

- Mermaid
- PlantUML
- 伪泳道流程图源码

这些内容虽然能表达逻辑，但在正式管理文档、培训材料、制度说明、流程宣导中，通常不够美观，也不够标准。

`swimlane-diagram` 的默认交付物不是代码，而是：

- 真正的 `SVG` 成品图

这意味着它更适合：

- 放进知识库
- 放进制度文档
- 放进流程手册
- 放进汇报材料
- 放进管理培训材料

也就是说，它更接近“正式业务图”的交付，而不是“技术临时图”。

### 2. 面向企业管理和流程制度场景专门优化

这个技能不是通用型画图工具，而是明显偏向企业管理场景。

它特别适合：

- 招聘管理流程
- 人事管理流程
- 合同管理流程
- 采购管理流程
- 资产管理流程
- 客户管理流程
- 线索管理流程
- 交付管理流程
- 售前 / 售后 / 验收流程
- 审批流程
- 跨部门协同流程

这些流程有一个共同特征：

- 不是简单的线性步骤
- 而是跨角色、跨部门、跨节点的协同链路

普通流程图能画“步骤”，但泳道图才能真正体现：

- 谁负责
- 谁协同
- 谁审批
- 在哪里发生交接
- 在哪里出现分支或异常

这也是它对企业管理更有价值的原因。

### 3. 视觉目标更接近 draw.io / 正式业务流程图

这个技能明确避免两种常见低质量输出：

- 像程序员看的技术图
- 像能渲染但不好看的草图

它追求的是一种更适合业务场景的视觉效果：

- 纵向泳道布局
- 顶部泳道标题栏
- 节点对齐规整
- 主路径稳定向下
- 决策节点清楚
- 分支尽量简洁
- 箭头落点准确
- 连线尽量少折角
- 颜色克制统一
- 留白充足，便于快速阅读

这类风格的好处是：

- 管理层看得懂
- 一线执行人员也看得懂
- 在文档里显得专业
- 不会像“技术人员自用图”那样过于随意

### 4. 内置严格的版式和质量约束

这个技能不是“随缘出图”。

它对图形质量做了明确约束，例如：

- 同类动作框必须等宽
- 关键节点应尽量同行对齐
- 文字必须居中
- 箭头必须落在节点边界，而不是扎进节点内部
- 分支标签必须避开节点和连线
- 连线不能压字
- 主流程尽量少折角
- 数据库 / 数据池图标必须符合标准圆柱形态
- 不允许拿 Mermaid 分组图伪装成正式泳道图

这些约束看起来细，但正是这些细节，决定了一张图是“像样”，还是“看起来很业余”。

### 5. 兼顾成品图与可编辑源码

虽然它默认产出 SVG 成品图，但并不意味着放弃可编辑性。

当用户明确要求时，它仍然支持：

- `PlantUML`
- `Mermaid`

其中：

- `PlantUML` 更适合需要结构化源码的场景
- `Mermaid` 只作为 Obsidian 原生兼容方案保留

这个设计保证了两件事：

- 正式交付时，优先保证图形质量
- 日常知识库维护时，仍然保留源码兼容能力

### 6. 适合长期构建统一的流程图体系

这个技能不只是为“一张图”服务。

它更适合拿来长期搭一整套流程图体系。

比如你可以持续输出：

- 行政管理流程集
- 商务管理流程集
- 培训管理流程集
- 服务管理流程集
- 营销管理流程集
- 产品管理流程集

因为它已经固定了模板、风格和规则，所以越往后用，图的风格越统一。

这对于企业来说非常重要。

因为真正有价值的，不是一张图画得好，而是整套流程图体系看起来像同一个系统，而不是不同时间、不同人、不同工具拼出来的碎片。

## 技能内包含的能力

当前仓库中的 `swimlane-diagram` 技能包含以下组成部分：

- `SKILL.md`
  核心规则文件，定义触发条件、默认输出形式、输出约束和边界
- `agents/openai.yaml`
  UI 元数据，方便技能在系统中被正确识别和调用
- `references/svg-swimlane-style.md`
  SVG 输出的视觉规范
- `references/enterprise-swimlane-template.md`
  企业管理类泳道图专用模板规则
- `references/bpmn-like-layout-rules.md`
  用于提升业务流程图的规范感和专业感
- `references/plantuml-swimlane-patterns.md`
  PlantUML 兼容输出规则
- `references/mermaid-swimlane-patterns.md`
  Mermaid 降级兼容说明
- `assets/enterprise-swimlane-skeleton.svg`
  企业管理流程图默认骨架模板

这意味着它不是一段简单提示词，而是一套真正可复用的技能结构。

## 适合哪些人使用

这个技能尤其适合以下几类人：

- 企业管理者
- 组织发展 / 流程设计人员
- 咨询顾问
- 飞书 / Base / CRM / ERP 流程落地人员
- 项目交付负责人
- 需要把制度和流程沉淀到知识库的人
- 想把流程标准化、模板化、可复制化的人

如果你的工作里经常出现以下需求，这个技能会特别有价值：

- “把这段制度整理成流程图”
- “把这个审批流画出来”
- “把这个交付过程做成泳道图”
- “把这个 SOP 变成一张团队能看懂的图”
- “把企业流程体系做成统一风格的图集”

## 推荐使用方式

安装技能后，可以直接这样使用：

- `用 $swimlane-diagram 把这个流程做成标准 SVG 泳道图`
- `用 $swimlane-diagram 按企业管理模板出一张正式流程图`
- `用 $swimlane-diagram 参考 draw.io 风格生成成品图`
- `用 $swimlane-diagram 把这段 SOP 转成泳道图`

如果你明确希望输出源码，也可以这样说：

- `用 $swimlane-diagram 输出 PlantUML 版本`
- `用 $swimlane-diagram 输出 Mermaid 版本`

## 这个技能的真正价值

`swimlane-diagram` 的价值，不在于“它会画图”，而在于：

- 它默认输出的是正式图，不是草图
- 它适合企业管理和跨部门协同这种高频业务场景
- 它内置了严格的图形质量控制规则
- 它更接近真实业务流程图，而不是技术图
- 它可以长期沉淀成统一的流程图体系

对于企业管理、制度建设、组织协同、流程设计、交付运营这类工作来说，这种能力的实用性远高于普通的 Mermaid 生成方案。

## 总结

如果你只是想临时表达一个简单流程，很多工具都能做到。

但如果你的目标是：

- 输出正式的泳道图成品
- 让图真正适合企业管理场景
- 长期沉淀统一风格的流程体系
- 让不同线程、不同上下文下的出图都尽量稳定

那么 `swimlane-diagram` 会比普通流程图技能更合适。

它不是“随便生成图”的工具，而是一套面向正式业务流程表达的标准技能。
