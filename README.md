# Skill

一个面向真实业务场景的 Codex 技能仓库。

当前核心技能：

- [`swimlane-diagram`](./swimlane-diagram)

这个技能专门解决一个高频但长期被低质量方案占据的问题：

**如何把企业管理、审批协同、交付运营、SOP 文档里的流程，真正做成一张标准、清晰、正式、能直接使用的泳道图。**

---

## 为什么会做这个技能

很多人都遇到过同一个问题：

你有流程，但流程只存在于文档里；  
你有逻辑，但逻辑没有图；  
你有图，但图不够正式；  
你有 Mermaid，但 Mermaid 看起来像草稿，不像真正能放进制度、培训、汇报和知识库里的流程图。

普通流程图方案通常有几个明显问题：

- 只适合表达逻辑，不适合正式展示
- 图能看，但不够专业
- 角色边界不清晰
- 跨部门交接看不明白
- 线条混乱，节点不对齐
- 生成的是“像图的代码”，不是“真正的图”

`swimlane-diagram` 就是为了解决这些问题而做的。

它的目标不是“把流程随便画出来”，而是：

- 把业务流程做成真正可交付的图
- 默认输出成品级 `SVG`
- 更接近 draw.io / 正式业务流程图风格
- 更适合企业管理、SOP、制度建设、跨部门协同这些真实工作场景

---

## 这个技能是什么

`swimlane-diagram` 是一个专门用于生成**标准泳道图**的 Codex 技能。

它适合把以下内容转成正式流程图：

- 企业管理流程
- SOP / 操作规范
- 审批流
- 合同流
- 采购流
- 交付流
- 客户流程
- 售前 / 售后 / 验收流程
- 各类跨部门协同流程

它不是“流程图源码生成器”，而是**流程图成品生成技能**。

默认输出：

- 高质量 `SVG` 泳道图

仅在用户明确要求时，才会输出：

- `PlantUML`
- `Mermaid`

也就是说，它默认把“图”当成交付物，而不是把“代码”当成交付物。

---

## 核心优势

### 1. 默认输出 SVG 成品图，而不是 Mermaid 草图

很多流程图技能的默认输出其实只是源码：

- Mermaid
- PlantUML
- 伪泳道图结构

这些内容可以表达逻辑，但在正式管理场景里往往不够好看，也不够标准。

`swimlane-diagram` 默认输出的是：

- 真正的 `SVG` 图形
- 可用于制度文档、SOP、培训材料、知识库、汇报材料的成品图

这使它更适合“真实业务交付”，而不是只适合笔记里的临时记录。

---

### 2. 面向企业管理和流程制度场景专门优化

这个技能不是通用画图器，而是明确偏向企业管理流程。

特别适合：

- 行政管理
- 商务管理
- 培训管理
- 服务管理
- 营销管理
- 产品管理
- 审批机制
- 跨部门协同机制

这些流程的共同特点是：

- 角色多
- 交接多
- 分支多
- 需要体现责任边界

而泳道图正是这类问题最合适的表达方式。

---

### 3. 输出风格更接近正式业务图，而不是技术图

这个技能明确追求一种更适合管理文档的视觉结果：

- 纵向泳道布局
- 顶部泳道标题栏
- 主流程方向清楚
- 节点规整对齐
- 决策节点明确
- 连线简洁克制
- 留白充足，阅读负担低

它不是那种“程序员能看懂，但管理层不愿意看”的图。

它更适合：

- 流程说明
- 管理制度
- 培训宣导
- 内部手册
- 汇报展示

---

### 4. 内置严格的图形质量约束

`swimlane-diagram` 不是随缘出图。

它对图形质量做了明确限制，例如：

- 同类动作框要等宽
- 关键节点尽量同行对齐
- 文字必须居中
- 箭头要贴边，而不是扎进节点内部
- 分支标签要避开线和文字
- 连线尽量少折角
- 主路径要稳定
- 数据池 / 数据库图标必须规范
- 不能拿 Mermaid 分组图伪装成正式泳道图

这些细节直接决定一张图到底是“专业”，还是“只是能看”。

---

### 5. 兼顾正式交付与兼容输出

虽然默认输出是 `SVG` 成品图，但这个技能并没有放弃可编辑性。

当用户明确要求时，它仍然支持：

- `PlantUML`
- `Mermaid`

这意味着它可以同时满足两类需求：

- 正式交付时，优先保证图形质量
- 日常维护时，仍保留结构化源码兼容方案

---

### 6. 适合长期沉淀成统一风格的流程图体系

这个技能不是只适合“一次性画一张图”。

它更适合长期搭建一整套流程图体系，例如：

- 招聘流程图集
- 合同流程图集
- 采购流程图集
- 客户流程图集
- 交付流程图集
- 售后流程图集

因为它已经固定了输出模板和视觉规则，所以越往后用，整体风格越统一。

这对企业来说很重要。

真正有价值的不是一张图画得好，而是整套流程体系看起来像同一个系统，而不是不同工具、不同时间、不同人随手拼出来的碎片。

## 使用方式

安装完成后，可以直接这样调用：

- `用 $swimlane-diagram 把这个流程做成标准 SVG 泳道图`
- `用 $swimlane-diagram 按企业管理模板生成成品图`
- `用 $swimlane-diagram 参考 draw.io 风格出图`
- `用 $swimlane-diagram 把这段 SOP 转成正式泳道图`

如果你明确想要源码而不是成品图，也可以这样说：

- `用 $swimlane-diagram 输出 PlantUML 版本`
- `用 $swimlane-diagram 输出 Mermaid 版本`
