# swimlane-diagram 详细介绍

`swimlane-diagram` 解决的是一个很常见但经常做得不够好的问题：如何把企业管理、审批协同、交付运营和 SOP 文档里的流程，整理成真正标准、清晰、正式、可直接使用的泳道图。

很多常见方案只能表达逻辑，不能直接形成专业成品。常见问题包括：

- 图能看，但不够正式
- 角色边界不清晰
- 节点不对齐
- 连线混乱
- 只能得到 Mermaid 或 PlantUML 源码

这个技能的设计重点不是“把流程随便画出来”，而是确保交付物足够规范，能直接进入制度文档、知识库、培训材料和汇报场景。

## 核心价值

- 默认交付高质量 `SVG` 成品图
- 输出风格更接近正式业务流程图
- 面向企业管理和跨部门协同场景优化
- 同时保留源码兼容输出能力

## 为什么适合企业管理

企业管理流程通常有几个共性：

- 角色多
- 交接多
- 分支多
- 责任边界需要清楚表达

泳道图是这类流程最合适的表达方式，但前提是图本身要足够规范。这个技能内置了布局和样式约束，避免把 Mermaid 分组图伪装成正式泳道图。
